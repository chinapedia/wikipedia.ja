**デッカーのアルゴリズム**は[オランダ](https://ja.wikipedia.org/wiki/オランダ "wikilink")人[数学者](../Page/数学者.md "wikilink") T・J・デッカーの考案した[相互排他のための](https://ja.wikipedia.org/wiki/ミューテックス "wikilink")[アルゴリズム](../Page/アルゴリズム.md "wikilink")である。これにより、[共有メモリ](../Page/共有メモリ.md "wikilink")による通信のみで、2つのプロセスが1つの[リソース](https://ja.wikipedia.org/wiki/リソース "wikilink")を競合することなく共有することができる。

厳密に交互にとっていく素朴なアルゴリズムを避けて発明された世界初の相互排他アルゴリズムの1つである。

ふたつのプロセスが同時に同じ[クリティカルセクション](https://ja.wikipedia.org/wiki/クリティカルセクション "wikilink")にアクセスしようとしたとき、このアルゴリズムはどちらのプロセスがアクセス権を得るかを決定する。もしもう一方のプロセスが既にクリティカルセクションに変更を加えていたら、その完了を待つ。

` f0 := false`
` f1 := false`
` turn := 0   // or 1`

` p0: f0 := true                      p1: f1 := true`
`     `**`while`**` f1 = true {                   `**`while`**` f0 = true {`
`         `**`if`**` turn ≠ 0 {                      `**`if`**` turn ≠ 1 {`
`             f0 := false                         f1 := false`
`             `**`while`**` turn ≠ 0 {                   `**`while`**` turn ≠ 1 {`
`             }                                   }`
`             f0 := true                          f1 := true`
`         }                                   }`
`     }                                   }`

`     // Start of Critical Section `
`       ・・・            `
`     // End of Critical Section `
`    `
`     turn := 1                           turn := 0`
`     f0 := false                         f1 := false`

## 注意事項

最近の多くのCPUは命令を[アウト・オブ・オーダー実行](../Page/アウト・オブ・オーダー実行.md "wikilink")する。このアルゴリズムは、そのようなCPUを使用した[マルチプロセッサ](https://ja.wikipedia.org/wiki/マルチプロセッサ "wikilink")マシンでは[メモリバリア](https://ja.wikipedia.org/wiki/メモリバリア "wikilink")がないと動作しない。

さらに、多くの最適化コンパイラはこのコードを変化させてしまい、プラットフォームがどうであれ、動作できなくなる。多くの言語では、フラグ変数 *f0* と *f1* がループ内で全くアクセスされないことを検出する。また多くのコンパイラは *turn* という変数が内側のループ内で更新されないことも検出して変換してしまい、結果として無限ループを形成してしまうだろう。このような変換をされると、このアルゴリズムはアーキテクチャに寄らず動作できなくなる。（訳注：[英語版では以上の記述が不正確であるとしてノートで議論されている](https://ja.wikipedia.org/wiki/:en:Dekker's_algorithm "wikilink")。具体的には最適化以前に変数がレジスタに置き換わってしまうのを妨げないと意味がないというもの）。
