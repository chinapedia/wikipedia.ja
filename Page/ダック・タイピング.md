**ダック・タイピング**（****）とは、[Smalltalk](../Page/Smalltalk.md "wikilink")、[Perl](../Page/Perl.md "wikilink")、[Python](../Page/Python.md "wikilink")、[Ruby](../Page/Ruby.md "wikilink")などのいくつかの[動的型付け](https://ja.wikipedia.org/wiki/動的型付け "wikilink")[オブジェクト指向](../Page/オブジェクト指向プログラミング.md "wikilink")[プログラミング言語](../Page/プログラミング言語.md "wikilink")に特徴的な[型付けの作法のことである](../Page/型システム.md "wikilink")。それらの言語では[オブジェクト](../Page/オブジェクト_\(プログラミング\).md "wikilink")（変数の[値](../Page/値_\(情報工学\).md "wikilink")）に何ができるかはオブジェクトそのものが決定する。これにより[ポリモーフィズム](../Page/ポリモーフィズム.md "wikilink")（多態性）を実現することができる。つまり、[静的型付け](https://ja.wikipedia.org/wiki/静的型付け "wikilink")言語である[Java](https://ja.wikipedia.org/wiki/Java "wikilink")や[C\#の概念で例えると](../Page/C_Sharp.md "wikilink")、オブジェクトがある[インタフェースのすべてのメソッドを持っているならば](https://ja.wikipedia.org/wiki/インタフェース_\(抽象型\) "wikilink")、たとえそのクラスがそのインタフェースを宣言的に実装していなくとも、オブジェクトはそのインタフェースを実行時に実装しているとみなせる、ということである。それはまた、同じインタフェースを実装するオブジェクト同士が、それぞれがどのような[継承階層を持っているのかということと無関係に](https://ja.wikipedia.org/wiki/継承_\(プログラミング\) "wikilink")、相互に交換可能であるという意味でもある。

この用語の名前は「[ダック・テスト](https://ja.wikipedia.org/wiki/ダック・テスト "wikilink")」に由来する。

> "If it walks like a duck and quacks like a duck, it must be a duck"

（もしもそれがアヒルのように歩き、アヒルのように鳴くのなら、それはアヒルに違いない）

[デーブ・トーマス](https://ja.wikipedia.org/wiki/デーブ・トーマス "wikilink")はRuby[コミュニティで初めてこの言葉を使ったと考えられている](../Page/インターネットコミュニティ.md "wikilink")。

[C++](../Page/C++.md "wikilink")の[テンプレートはダック](../Page/テンプレート_\(プログラミング\).md "wikilink")・タイピングの静的版である。例えば、[STLにおける各種のiteratorはIterator基底クラスのようなものからメソッドを継承しているわけではないが](https://ja.wikipedia.org/wiki/Standard_Template_Library "wikilink")、同じ構文でコンパイルが通るならば、それはiteratorの一種として扱える。この文脈で言う「同じインタフェースを持つ」とは、コンパイラにとって[シグネチャ](https://ja.wikipedia.org/wiki/シグネチャ "wikilink")などのインタフェースが同じだということである。したがって、iteratorの実装はオブジェクトである必要すらない。

もう1つ、ダック・タイピングに似たアプローチに[OCaml](../Page/OCaml.md "wikilink")のstructural subtypingがある。メソッドのシグネチャが互換ならば、宣言上の継承関係は関係なくとも、オブジェクトの型は互換であるというものである。これはOCamlの[型推論](https://ja.wikipedia.org/wiki/型推論 "wikilink")システムによってコンパイル時にすべて決定される。

## Rubyでの例

Rubyでの単純な例を示す。

``` ruby
 def test(foo)
   puts foo.sound
 end

 class Duck
   def sound
     'quack'
   end
 end

 class Cat
   def sound
     'myaa'
   end
 end

 test(Duck.new)
 test(Cat.new)
```

出力は以下である。

`quack`
`myaa`

2つのクラスに継承の関係が無いことに注目して欲しい。

## C++での例

上記Rubyの例をC++で記述すると、以下のようになる。

``` cpp
 #include <iostream>

 template <class T>
 void test(const T& t) {
   std::cout << t.sound() << std::endl;
 }

 struct Duck {
   const char* sound() const {
     return "quack";
   }
 };

 struct Cat {
   const char* sound() const {
     return "myaa";
   }
 };

 int main() {
   test(Duck());
   test(Cat());
 }
```

実行結果はRubyの例と同じである。ただし、テンプレートによるダック・タイピングはコンパイル時に解決される静的なポリモーフィズムであり、動的型付け言語とは異なり実行時のオーバーヘッドを伴わない。

## C\#での例

C\#はバージョン4.0で動的型付けを可能にする`dynamic`型が使えるようになった。`dynamic`は内部的には[リフレクションを利用して実装されており](../Page/リフレクション_\(情報工学\).md "wikilink")、該当するメソッドやプロパティの存在有無を実行時に遅延評価する。

``` csharp
using System;

class Duck {
    public string Sound() {
        return "quack";
    }
}

class Cat {
    public string Sound() {
        return "myaa";
    }
}

public class DuckTypingTest {
    static void Test(dynamic obj) {
        Console.WriteLine(obj.Sound());
    }

    public static void Main() {
        Test(new Duck());
        Test(new Cat());
    }
}
```

なお、C\#はC++のテンプレートに似た機能として[ジェネリクス](https://ja.wikipedia.org/wiki/ジェネリクス "wikilink")をサポートするが、C++テンプレートほどの柔軟性はなく、ダック・タイピングに使用することはできない\[1\]。

## その他の言語など

Javaは言語構文レベルで動的型付けをサポートしないが、リフレクションを用いることで、ダック・タイピング相当を実現できる。

[COMにおいても](../Page/Component_Object_Model.md "wikilink")、`IDispatch`インタフェース\[2\]を実装することで、ダック・タイピング相当を実現できる。
