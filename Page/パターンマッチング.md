**パターンマッチング** (Pattern matching、パターン照合) とは、データを[検索](../Page/検索.md "wikilink")する場合に、特定のパターンが出現するかどうか、またどこに出現するかを特定する手法のことである。

文字列のパターンマッチングには、固定されたパターンの検索では[KMP法や](https://ja.wikipedia.org/wiki/クヌース–モリス–プラット法 "wikilink")[BM法など各種の](https://ja.wikipedia.org/wiki/ボイヤー-ムーア文字列検索アルゴリズム "wikilink")[文字列探索](https://ja.wikipedia.org/wiki/文字列探索 "wikilink")アルゴリズムがある。また[正規表現](../Page/正規表現.md "wikilink")を利用する手法も多数提案されている。

画像や動画に対するパターンマッチングの研究も行われている。だが、パターンマッチングはあらかじめ人が打っておかなくてはいけないため人工知能とは別で機械が自分で考えているわけではない（そもそも「考える」ということを形式的に定義することは不可能なので、この段落の後半の「だが、」以降は、単にどこかの誰かの考える「人工知能」という語に関する主観の表明に過ぎず、意味があることを何も述べてはいない）。

いくつかの[高水準プログラミング言語には](https://ja.wikipedia.org/wiki/高水準言語 "wikilink")、多分岐の一種で、場合分けと同時に構成要素の取り出しのできる言語機能があり、パターンマッチと呼ばれている。[Haskell](../Page/Haskell.md "wikilink")での例を示す。

``` haskell
listSumCase lst =
  case lst of
    []       -> 0
    (x : xs) -> x + listSumCase xs

listSumPtn []       = 0
listSumPtn (x : xs) = x + listSumPtn xs
```

なお、この例の `listSumCase` は expression style、`listSumPtn` は declaration style である\[1\]。
