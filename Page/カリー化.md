**カリー化** (currying, カリー化された=curried) とは、複数の[引数](../Page/引数.md "wikilink")をとる[関数を](https://ja.wikipedia.org/wiki/関数_\(プログラミング\) "wikilink")、引数が「もとの関数の最初の引数」で戻り値が「もとの関数の残りの引数を取り結果を返す関数」であるような関数にすること（あるいはその関数のこと）である。[クリストファー・ストレイチー](https://ja.wikipedia.org/wiki/クリストファー・ストレイチー "wikilink")により論理学者[ハスケル・カリー](../Page/ハスケル・カリー.md "wikilink")にちなんで名付けられたが、実際に考案したのはMoses Schönfinkelと[ゴットロープ・フレーゲ](../Page/ゴットロープ・フレーゲ.md "wikilink")である。

ごく簡単な例として、f(a, b) = c という関数 f があるときに、F(a) = g（ここで、g は g(b) = c となる関数である）という関数 F が、f のカリー化である。

関数 *f* が \(f:(X \times Y) \to Z\) の形のとき、\(f\) をカリー化したものを \(g\) とすると、\(g:X \to (Y \to Z)\) の形を取る。uncurryingは、これの逆の変換である。

[理論計算機科学](../Page/理論計算機科学.md "wikilink")の分野では、カリー化を利用すると、複数の引数をとる関数を、一つの引数のみを取る複数の関数の[ラムダ計算](../Page/ラムダ計算.md "wikilink")などの単純な理論的モデルと見なして研究できるようになる。[圏論](../Page/圏論.md "wikilink")ではカリー化の概念を、[デカルト閉圏](https://ja.wikipedia.org/wiki/デカルト閉圏 "wikilink")における[冪対象](https://ja.wikipedia.org/wiki/冪対象 "wikilink")の[普遍性](../Page/普遍性.md "wikilink")に見出せる。適当な2つの[対象の](https://ja.wikipedia.org/wiki/対象_\(圏論\) "wikilink")[積から別の対象への](https://ja.wikipedia.org/wiki/積_\(圏論\) "wikilink")[射](../Page/射_\(圏論\).md "wikilink") \(f: X \times Y \to Z\) に対して、射 \(g: X \to Z^Y\) が一意に対応する。

カリー化をする現実の動機の1つに、カリー化することで後述する[部分適用](https://ja.wikipedia.org/wiki/部分適用 "wikilink")が行いやすくなることが挙げられる。たとえば、加算を行う関数 `(+)` をカリー化してから、最初の引数だけに `1` を適用すれば、インクリメント用の関数が簡単に作れる。

カリー化を基盤としている[プログラミング言語](../Page/プログラミング言語.md "wikilink")もある。特に[MLと](../Page/ML_\(プログラミング言語\).md "wikilink")[Haskell](../Page/Haskell.md "wikilink")では関数は常に一つの引数のみを取り、複数の引数を取る関数とは、単にネストされた複数の一引数関数の[糖衣構文](../Page/糖衣構文.md "wikilink")にすぎない。[第一級関数](https://ja.wikipedia.org/wiki/第一級関数 "wikilink")を扱える言語、たとえば[LISP](https://ja.wikipedia.org/wiki/LISP "wikilink")、[Scheme](../Page/Scheme.md "wikilink")、[F\#](../Page/F_Sharp.md "wikilink")、[Scala](../Page/Scala.md "wikilink")、[Erlang](../Page/Erlang.md "wikilink")、[Eiffel](../Page/Eiffel.md "wikilink")、[Perl](../Page/Perl.md "wikilink")、[Ruby](../Page/Ruby.md "wikilink")、[Python](../Page/Python.md "wikilink")、[R言語](https://ja.wikipedia.org/wiki/R言語 "wikilink")、[S言語](https://ja.wikipedia.org/wiki/S言語 "wikilink")、[JavaScript](../Page/JavaScript.md "wikilink")、[Swiftなどでは](https://ja.wikipedia.org/wiki/Swift_\(プログラミング言語\) "wikilink")、カリー化関数を作ることができる。

## 実装例

たとえば、除算の関数 \(\mathrm{div}(x, y) = x / y\) をカリー化したものを \(\mathrm{cdiv}(x)\) とすると、\(\mathrm{cdiv}(x)\) は \(x\) のみを引数に取る。そして \(\mathrm{inv} = \mathrm{cdiv}(1)\) とすると、\(\mathrm{inv}(y)\) は \(y\) のみを引数に取る新しい関数となり、\(\mathrm{inv}(y) = 1 / y\)、つまり引数の逆数を返す関数になる。

この例をSchemeとJavaScriptで実装した例を示す。

``` scheme
(define div (lambda (x y) (/ x y)))
;(div 1 3) == (/ 1 3), これはカリー化ではない。

(define cdiv (lambda (x) (lambda (y) (div x y) )))
;(cdiv a) == (div a y)
((cdiv 10) 3)
;=> 10/3

(define inv (lambda (x) ((cdiv 1) x)))
;inv == (cdiv 1)
(inv 2)
;=> 1/2
```

``` javascript
function div(x, y) { return x / y; }
//div(1, 3) == 1 / 3, これはカリー化ではない。

function cdiv(x) { return function(y) { return div(x, y); } }
console.log(cdiv(10)(3));
//=> 10/3 = 3.333...

var inv = cdiv(1);
console.log(inv(2));
//=> 1/2 = 0.5
```

## 部分適用との混同

カリー化は[部分適用](https://ja.wikipedia.org/wiki/部分適用 "wikilink")と混同されやすい\[1\]\[2\]。部分適用とは、複数の引数をとる関数の一部の引数に実引数を適用する操作のことで、例えば上述の \(\mathrm{div}(x, y)\) から \(\mathrm{inv}(y)\) を導く操作を指す。一方、カリー化は例えば \(\mathrm{div}(x, y)\) から \(\mathrm{cdiv}(x)\) を導く操作であり、引数への実引数の適用までは行わない。

[標準C++ライブラリ](../Page/標準C++ライブラリ.md "wikilink")の `std::bind1st` は部分適用の一例である。

この混同はしばしば言語設計者によってもなされる。[Groovy](https://ja.wikipedia.org/wiki/Groovy "wikilink") では、部分適用を行う標準メソッドに `curry` という名前がつけられている。また、[Haskell](../Page/Haskell.md "wikilink") の標準関数 `curry` も、カリー化と厳密には違う動作をする。この関数は `(a, b) -> c` という型を持つ関数を `a -> b -> c` 型の関数に変換するが、元の関数は「2 要素の[タプル](https://ja.wikipedia.org/wiki/タプル "wikilink")を取る 1 引数関数」であり、2 引数関数ではない。Haskell は全ての関数が 1 引数関数であり（つまり、全ての関数が元々カリー化された形であるとも表現される）、Haskell プログラムでは、厳密な意味でカリー化を行う関数は定義できない。ただし型理論上は、`X`, `Y` という型の要素を持つタプルの型は直積 \(X \times Y\) とほぼ同一視できるため、必ずしも誤用というわけではない。
