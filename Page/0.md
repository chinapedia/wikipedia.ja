[thumb](https://ja.wikipedia.org/wiki/ファイル:零-order.gif "wikilink") 文字 **0** によって表されるものは、[何もないことに対応する](../Page/無.md "wikilink")[基数](../Page/基数詞.md "wikilink")（[自然数](../Page/自然数.md "wikilink")\[1\]）であり、[1](https://ja.wikipedia.org/wiki/1 "wikilink") の直前の[序数](../Page/序数詞.md "wikilink")（[順序数](../Page/順序数.md "wikilink")）であって、最小の[非負整数である](https://ja.wikipedia.org/wiki/正の数と負の数 "wikilink")。また、[−1](../Page/−1.md "wikilink") の次の整数でもある。**零**（れい、ぜろ）、**ゼロ**（）、**セロ**（）**ヌル**（）、**ノート**（）、**ニヒル**（）などと読まれる。また、文字の形状から、稀に[まる](https://ja.wikipedia.org/wiki/まる "wikilink")あるいは[オーなどのように呼ばれることもある](https://ja.wikipedia.org/wiki/O "wikilink")。なお、日本の[通話表においては](https://ja.wikipedia.org/wiki/通話表#和文通話表 "wikilink")、0 は「数字の**まる**」と送られる。

数としての 0 は、[整数](../Page/整数.md "wikilink")全体、[実数](../Page/実数.md "wikilink")全体（あるいはもっと一般の数からなる代数系で）[加法単位元](https://ja.wikipedia.org/wiki/加法単位元 "wikilink")としての役割を演じる。文字としての 0 の使用は[位取りによる記数法における](https://ja.wikipedia.org/wiki/位取り記数法 "wikilink")[プレースホルダとして有用である](../Page/メタ構文変数.md "wikilink")。

## 数としての 0

**0** は [1](https://ja.wikipedia.org/wiki/1 "wikilink") の直前の整数である。多くの数体系で 0 は負の概念よりも前に同定され、負の概念は 0 よりも小さいものとして理解される。0 は[偶数](../Page/偶数.md "wikilink")である\[2\]\[3\]。0 は正の数でも負の数でもない。0 を[自然数](../Page/自然数.md "wikilink")とする定義もあり、その場合自然数と正の整数は同義ではない。

0は数量が[空っぽであることを意味する数である](../Page/空集合.md "wikilink")。兄弟が0人いるというのは兄弟がひとりも居ないことを意味し、重さが0であるというのは重さが無いことを表す。あるいは二つの砂山の砂粒の数の差が0であるということは、その二つの砂粒の数の差がないことを意味する。

数を数えはじめるまえは、ものが0個であると仮定することができる。つまり、最初のものを数え始めるまでは0で、最初のものを持ってきてはじめて1であると勘定することになる。ほとんどの[歴史学者](https://ja.wikipedia.org/wiki/歴史学者 "wikilink")をはじめ世界中の人々は[グレゴリオ暦](../Page/グレゴリオ暦.md "wikilink")や[ユリウス暦](../Page/ユリウス暦.md "wikilink")から[紀元0年](https://ja.wikipedia.org/wiki/紀元0年 "wikilink")を除いて考えるが、[天文学者](../Page/天文学者.md "wikilink")などは計算上不都合があるため暦に紀元0年を含めて考える。また、（紀元）0年という文言は、[時間](../Page/時間.md "wikilink")における新しい起点となりうる、非常に意義深い出来事を記述する場合にも用いられることがある。

## 数字としての 0

[71px](https://ja.wikipedia.org/wiki/ファイル:Text_figures_036.svg "wikilink") 現代的な数字の 0 は、円、楕円、角の丸い長方形のような形に書かれるのが普通である。最も現代的な[書体](../Page/書体.md "wikilink")では 0 は他の数字と高さが同じになるものが普通だが、のある書体では、左図のように0の文字の高さが低くなることが多い（）。

[位取り記数法](https://ja.wikipedia.org/wiki/位取り記数法 "wikilink")で用いられる数字の 0 は、数あるいは数値としての 0 とは別物である。位取り記数法における数字の並びは上位の桁の数字がより高い重みを持つので、位取り記数法における数字の 0 は空位を表すのに用いられ、それによって下位および上位の桁の数字に適切な重みを与えることができる。位取り記数法で数字の 0 がいつでも必要というわけではない。たとえば 02 は数としては 2 と同値であるため先頭の 0 は冗長である。

稀に、頭に 0 を付けた数値を付いていない数値と別のものとして扱うことがある。例えば[ルーレット](../Page/ルーレット.md "wikilink")で '00' は '0' とは別（'0' に賭けたなら玉が '00' に止まっても勝ちにならないし、逆もそう）である。競技者に番号が振られるスポーツなども同様で、例えば[ストックカー](../Page/ストックカー.md "wikilink")で '07' 番の車は '7' 番の車とは別だと看做される。

### 数字の 0 とアルファベットの O との区別

[数字 0 とアルファベット O との字形の比較](https://ja.wikipedia.org/wiki/ファイル:Zero_o_comparison.svg "wikilink")

伝統的に、多くの印刷書体では、[似た字体である](../Page/ホモグリフ.md "wikilink")0と大文字の[O](https://ja.wikipedia.org/wiki/O "wikilink")（オー）との区別のために、大文字の O を細い楕円形の 0 よりもさらに丸いものにしている\[4\]。[タイプライター](../Page/タイプライター.md "wikilink")ではもともと O と 0 の字形を区別してはいなかったし、0 に対してキーを割り当てていないモデルすら存在した。これらの字形に区別がはっきりと生じるのは、コンピュータのプログラムのように明確に識別できることが必要になった\[5\]、現代的な文字[表示装置においてである](../Page/ディスプレイ_\(コンピュータ\).md "wikilink")\[6\]。

中央に点のある 0 が用いられたのは [IBM 3270](../Page/IBM_3270.md "wikilink") 表示装置の付属文字が最初であろう。この字体は [Microsoft Windows](https://ja.wikipedia.org/wiki/Microsoft_Windows "wikilink") でも [Andalé Mono](https://ja.wikipedia.org/wiki/Andalé_Mono "wikilink") 書体に受け継がれている。点の代わりに短い縦棒を用いたものもあり、これは解像度の悪い表示画面では[ギリシャ文字](https://ja.wikipedia.org/wiki/ギリシャ文字 "wikilink")の [Θ](../Page/Θ.md "wikilink") と紛らわしいかもしれないが、Θ が表示可能な文字でなかったりともかくあまり使われないなどの理由で現実的にはそれほど問題となってはいない。

他に、[斜線付きゼロ](https://ja.wikipedia.org/wiki/斜線付きゼロ "wikilink")（O を / で串刺しにしたような字形）が初めて用いられたのは、パンチカードやテープに転写する前の手書きコーディングシートにおいてであり、[ASR-33](../Page/ASR-33.md "wikilink") テレタイプのデフォルトのタイプホイールの流れを汲む旧式の [ASCII](../Page/ASCII.md "wikilink") [図形文字](https://ja.wikipedia.org/wiki/図形文字 "wikilink")集合においても用いられる。この字形は[空集合](../Page/空集合.md "wikilink")を表す記号 \(\scriptstyle\emptyset\) あるいは "∅"（[Unicode](../Page/Unicode.md "wikilink")で U+2205 の文字）や、いくつかの[スカンジナビア語群で用いられる](../Page/北ゲルマン語群.md "wikilink") [Ø](../Page/Ø.md "wikilink") とも似ている。[CSSでは](../Page/Cascading_Style_Sheets.md "wikilink")、[OpenType](../Page/OpenType.md "wikilink")フォントのfont-featureタグを用いて、0に斜線を入れることもできる\[7\]。Unicodeでは、バージョン9.0にて[異体字セレクタ](../Page/異体字セレクタ.md "wikilink")を用いたシーケンスU+0030（数字0） U+FE00（VS1）で斜線付き0が定義された。

逆に、文字 O に斜線をつけて数字の 0 につけない慣習を支持するのが、著名な[IBM](../Page/IBM.md "wikilink")ユーザーグループの SHARE であり\[8\]、[FORTRAN](https://ja.wikipedia.org/wiki/Fortran "wikilink") のプログラムの書式としてこの慣習が IBM によって推奨されている\[9\]。また、他のいくつかの初期のメインフレームメーカーもこれを支持している。この慣習は[スカンジナビア人](https://ja.wikipedia.org/wiki/スカンジナビア人 "wikilink")にとっては二つの文字の衝突を意味するため十分問題含みである。これらの他は、IBM の [Algol](https://ja.wikipedia.org/wiki/Algol "wikilink") プログラムの書式をも含め\[10\]、これとは逆の慣習を支持している\[11\]。 [バロース](../Page/バロース.md "wikilink")/[ユニシス](../Page/ユニシス.md "wikilink")の画面表示装置には逆斜線つき 0 を備えたものもある。別の慣習として、初期の[ラインプリンタでは飾りのない](https://ja.wikipedia.org/wiki/プリンター "wikilink") 0 を残したものの、大文字の O にはしっぽやひげをくわえて、逆向きの [Q](../Page/Q.md "wikilink") や[筆記体](../Page/筆記体.md "wikilink")の大文字 O (\(\scriptstyle\mathcal{O}\)) のように見える字形としたものがあった\[12\]。とあるタイプ式のプリンタで、0 と O は別の活字になっているのに、あまりに似ていて区別ができないデザインであることに腹を立てた人が、O の活字をヤスリで加工して切れ目を入れてしまった、というエピソードもある\[13\]。

計算機での使用を目的として設計されたフォントでは、アルファベット O と数字 0 の一方をより丸く他方をより（長方形に近く）角ばらせているものがある。[テキサス・インスツルメンツ](../Page/テキサス・インスツルメンツ.md "wikilink")の [TI-99/4A](https://ja.wikipedia.org/wiki/TI-99/4A "wikilink") 計算機では大文字の O が四角くて数字の 0 が丸いという特徴であったが、これ以外の計算機では逆の選択がなされている。 [thumb](https://ja.wikipedia.org/wiki/ファイル:Deutsches_Kfz-Kennzeichen_für_Behördenfahrzeuge_\(Nummernbereich_3\).jpg "wikilink")

[ヨーロッパ](../Page/ヨーロッパ.md "wikilink")の大部分では、車輌の[ナンバープレート](../Page/ナンバープレート.md "wikilink")の書体でこの方法を部分的に用いて（0 を四角くしたり、0 よりも O のほうを幅広にしたりして）これらの記号を区別しているが、国によってはさらに 0 の右上隅に切込みをいれてより明確な区別をつけているものもある（たとえば、[ドイツの車輌ナンバープレート](https://ja.wikipedia.org/wiki/ドイツの車輌ナンバープレート "wikilink")で用いられている[変造防止文字](https://ja.wikipedia.org/wiki/変造防止文字 "wikilink")  など）。

時には、混乱を完全に避けるために専ら数字の 0 を用いたり逆にまったく用いなかったりすることもある。例えば、[サウスウエスト航空](../Page/サウスウエスト航空.md "wikilink")で用いられている予約番号では数字の 0 と 1 の代わりに専ら大文字の O と I が使用されている\[14\]し、反対に[カナダの郵便番号](https://ja.wikipedia.org/wiki/カナダの郵便番号 "wikilink")や[日本銀行券](../Page/日本銀行券.md "wikilink")の記番号\[15\]では 1 と 0 が用いられるのみで、大文字の I と O は使用されていない。

## 歴史

### 0 の起源

ゼロの発明は、[数学](../Page/数学.md "wikilink")史の飛躍の一つである。

紀元前2500年頃のピラミッドの[幾何学](../Page/幾何学.md "wikilink")的な正確性は、[古代エジプト](../Page/古代エジプト.md "wikilink")人が高度な数学を持っていたことを示す。しかし[古代エジプトでは数学は主に暦法と土地測量の手法として発展したため](https://ja.wikipedia.org/wiki/エジプト数学 "wikilink")、零の研究は発達せず、それを表す記号もなかった。面積がゼロの土地はなく、0日めのあるカレンダーもない。よってゼロは不要であった\[16\]。

紀元前500年頃、[楔形文字](../Page/楔形文字.md "wikilink")を使っていた[メソポタミア](../Page/メソポタミア.md "wikilink")文明で、位が 0 であることを示す文字を使い始めたことがわかっている。[六十進法](../Page/六十進法.md "wikilink")を用いた[バビロニアの数の表記には](https://ja.wikipedia.org/wiki/バビロニア数学 "wikilink")、古くは 0 がなく、例えば「62」と「602」は表記が全く同じで、見分けがつかなかった。これは非常に不便であったので、時代が下ると 6 と 2 の間に斜めの楔を並べて、62 と 602 の区別を表すようになった。この文字が人類が最初に 0 を表現した記号とされている。しかし、0 自体を数のうちとして扱ってはいなかった。

古代ギリシャでは高度な数学と天文学が発展したが、[ギリシャ数字](https://ja.wikipedia.org/wiki/ギリシャ数字 "wikilink")は計算が非常に面倒だったので、天文学のような大がかりな計算にはバビロニアから伝わった六十進法を使った。最も古い例は130年、[プトレマイオスが零と六十進法を用いて計算をした記録がある](../Page/クラウディオス・プトレマイオス.md "wikilink")。ギリシャの数学者はこの方法で時間や角度を計算し、1時間、また角度の1[度の](../Page/度_\(角度\).md "wikilink")1/60を1分、1/60分を1秒とした。しかし整数部分である1時間単位や角度そのものには零を使わず、12時の次は0時ではなく1時であった。ギリシャでは、バビロニアのゼロ記号には[ギリシア文字](../Page/ギリシア文字.md "wikilink")の[オミクロン](https://ja.wikipedia.org/wiki/オミクロン "wikilink")「ο」を当てた。アラビア数字の0と形が似ているが、これはおそらく偶然であったとされる\[17\]。

[アルキメデス](../Page/アルキメデス.md "wikilink")は「ある数とある数を足せば、結果は元の数より大きくなる」という「[アルキメデスの公理](https://ja.wikipedia.org/wiki/アルキメデスの公理 "wikilink")」を定立したが、足しても増えない性質を持つゼロは、この公理上、数ではないことになる\[18\]。古代ギリシア人は「ο」を単に[小数点](https://ja.wikipedia.org/wiki/小数点 "wikilink")のような[位取りを表す補助記号として使い](https://ja.wikipedia.org/wiki/位取り記数法 "wikilink")、数のうちに含めなかった。[ギリシア数字](https://ja.wikipedia.org/wiki/ギリシア数字 "wikilink")にはゼロを示す文字がなく、ギリシャの数体系を継承した[ローマ数字](../Page/ローマ数字.md "wikilink")にもゼロにあたる数字がない。

古代西洋で 0 の概念が受容されなかったのは、その宇宙観によるところが大きかった。[アリストテレス](../Page/アリストテレス.md "wikilink")は「自然は[真空](../Page/真空.md "wikilink")を嫌う」と宣言し、空間は必ず何らかの物質が充満しているとして真空、つまり「無」の存在を認めなかった。またアリストテレスは、宇宙を地球を中心にする球である[天球](../Page/天球.md "wikilink")と定義し、有限なものと考えた。この哲学からは「[無](../Page/無.md "wikilink")」と「[無限](../Page/無限.md "wikilink")」は認められなかった\[19\]。

アリストテレス哲学を源流とする「無」と「無限」を否定する宇宙観は[中世ヨーロッパ](https://ja.wikipedia.org/wiki/中世ヨーロッパ "wikilink")に継承され、宗教の一部と化した。17世紀まで、ヨーロッパでゼロや無限を主張することは、キリスト教への冒涜であり、死刑宣告を意味した。中世ヨーロッパではゼロを悪魔の数字とみなし、[ローマ法王](https://ja.wikipedia.org/wiki/ローマ法王 "wikilink")により使用が禁じられた\[20\]。1600年には、宇宙が無限であると主張した[修道士](../Page/修道士.md "wikilink")の[ジョルダーノ・ブルーノ](../Page/ジョルダーノ・ブルーノ.md "wikilink")が、[異端](../Page/異端.md "wikilink")の罪で火あぶりの刑にされている。

「無」が実在することを認め、ゼロを数として定義したのは「無」や「無限」を含む宇宙観を持ち、哲学的に「無」を追究した古代インドにおいてである。0の位置を記号で表わすバビロニアの方法はインドにも伝わった。最近になって[オックスフォード大学](../Page/オックスフォード大学.md "wikilink")の研究チームが、[1881年](../Page/1881年.md "wikilink")に現[パキスタン](../Page/パキスタン.md "wikilink")国内で発見された[バクシャーリー写本](https://ja.wikipedia.org/wiki/バクシャーリー写本 "wikilink")と呼ばれる[カバノキ](https://ja.wikipedia.org/wiki/カバノキ "wikilink")の樹皮の巻物の[数学書](https://ja.wikipedia.org/wiki/数学書 "wikilink")が、これまで考えられていたより500年古い[3](../Page/3世紀.md "wikilink") - [4世紀](../Page/4世紀.md "wikilink")頃のものであることを[年代測定](https://ja.wikipedia.org/wiki/年代測定 "wikilink")で特定した。そしてこの巻物に記された黒点が、インドにおける最古の0を表す文字であることになった\[21\]\[22\]。

古代[インドの数学](../Page/インドの数学.md "wikilink")で数としての「0」の概念が確立されたのは、はっきりしていないが5世紀頃とされている。数学者の[ブラーマグプタ](https://ja.wikipedia.org/wiki/ブラーマグプタ "wikilink")は、628年に著した『[ブラーマ・スプタ・シッダーンタ](../Page/ブラーマ・スプタ・シッダーンタ.md "wikilink")』において、0 と他の[整数](../Page/整数.md "wikilink")との[加減乗除](https://ja.wikipedia.org/wiki/加減乗除 "wikilink")を論じ、0 / 0 を 0 と定義した以外はすべて現代と同じ定義をしている。ゼロは「シューニャ」（ 。うつろな）と呼ばれた。9世紀頃の[グワリオールの寺院の壁に残る](../Page/グワーリヤル.md "wikilink")[碑銘](https://ja.wikipedia.org/wiki/碑銘 "wikilink")が「0」の最古の例とされていたが、[バクシャーリー写本](https://ja.wikipedia.org/wiki/バクシャーリー写本 "wikilink")の年代が[放射性炭素年代測定](../Page/放射性炭素年代測定.md "wikilink")によって3-4世紀であることが判明したため、年代が約500年さかのぼった\[23\]。これがアラビアに伝播し、[フワーリズミー](../Page/フワーリズミー.md "wikilink")の著作のラテン語訳  により「[アラビア数学](../Page/アラビア数学.md "wikilink")」の「0」として西欧に広まっていった。

インド・中東・ヨーロッパを除いて、数や空位を表す記号としての「0」を確立した地域としては、[マヤ文明](https://ja.wikipedia.org/wiki/マヤ文明 "wikilink")が代表例である。[マヤ数字](https://ja.wikipedia.org/wiki/マヤ数字 "wikilink")は[二十進法](../Page/二十進法.md "wikilink")を用いたが、「0」を意味する記号として貝殻に似た模様[MAYA-g-num-0-inc-v1.svg](https://ja.wikipedia.org/wiki/File:MAYA-g-num-0-inc-v1.svg "fig:MAYA-g-num-0-inc-v1.svg") が用いられた。二十は「点1つの下に貝殻模様」という[位取り記数法](https://ja.wikipedia.org/wiki/位取り記数法 "wikilink")であり、例えば十進表記の1616（二十進表記の40G）は上から順に「点(一)が4つ | 貝殻模様 | 横棒(五)が3つ + 点(一)が1つ」という形式で表記された。

中国では[算木](../Page/算木.md "wikilink")が紀元前から使われており、[位取り記数法](https://ja.wikipedia.org/wiki/位取り記数法 "wikilink")が確立していたが、空位は空白で表していた。算木を実際に使うときは誤解がないが、それを書写するときは紛らわしい。後に空位を「〇」と書くようになった。これはインドの「0」が輸入されたとも、元々、漢文で空白を表す「囗」が「〇」に変化したともいう。[漢数字\#〇、零](https://ja.wikipedia.org/wiki/漢数字#〇、零 "wikilink")を参照。

## 数学における 0 の使用

### 初等代数学

数の 0 は最小の非負整数である。0 の後続の[自然数](../Page/自然数.md "wikilink")は 1 であり、0 より前に自然数は存在しない。数 0 を自然数に含めることも含めないこともあるが、0 は整数であり、有理数であり、実数（あるいは代数的数、複素数）である。

数 0 は正でも負でもなく、[素数](../Page/素数.md "wikilink")でも[合成数](../Page/合成数.md "wikilink")でも[単数](https://ja.wikipedia.org/wiki/単数 "wikilink")でもない。一方、0 は偶数である。

以下は数 0 を扱う上での初等的な決まりごとである。これらの決まりは *x* を任意の実数あるいは複素数として適用して構わないが、それ以外の場合については何も言及していないということについては理解されなければならない。

  - 加法: *x* + 0 = 0 + *x* = *x*. つまり 0 は加法に関する[単位元](../Page/単位元.md "wikilink")である。
  - 減法: *x* − 0 = *x*, 0 − *x* = −*x*.
  - 乗法: *x* · 0 = 0 · *x* = 0.
  - 除法: *x* が 0 でなければ  = 0 である。しかし  は、0 が乗法に関する逆元を持たないために、（従前の規則の帰結としては）定義されない（[ゼロ除算](../Page/ゼロ除算.md "wikilink")を参照）。実数の範囲で考えるならば、正の数 *x* に対し、商  の *y* を 0 に正の側から近づけるならば、商の値は正の[無限大](https://ja.wikipedia.org/wiki/無限大 "wikilink")に向かって限りなく増加する。一方 *y* を負の側から 0 に近づければ、商の値は負の[無限大](https://ja.wikipedia.org/wiki/無限大 "wikilink")に向かって限りなく減少する。記号で書けば、
    <div style="margin:1ex auto 1ex 2em;">
    <math>\\begin{align}

x \> 0 &\\implies \\lim_{y \\to 0^+} {x \\over y} = +\\infty, \\\\ x \> 0 &\\implies \\lim_{y \\to 0^-} {x \\over y} = -\\infty \\end{align}</math>

</div>

が成立する。

  - 冪乗: *x* = 0 の場合にきちんと定義できないまま残される文脈があること（[0の0乗](../Page/0の0乗.md "wikilink")を参照）を除けば、*x*<sup>0</sup> = 1 である。任意の正の実数 *x* に対して 0<sup>*x*</sup> = 0 である。

  - なる式が、 の形の式の極限を決定しようとするなかで、それぞれ独立に分子分母の極限を取った結果として現れるかもしれない。これはと呼ばれる。これは単に必ずしも極限が求まらないということを意味するものではなく、むしろ  の極限は、それが存在するならば、[ロピタルの定理](../Page/ロピタルの定理.md "wikilink")のような別の方法によって求めるべきであるということを意味する。

  - [0 個の対象の和は](https://ja.wikipedia.org/wiki/空和 "wikilink") 0 であり、 [0 個の対象の積は](https://ja.wikipedia.org/wiki/空積 "wikilink") 1 である。[階乗](../Page/階乗.md "wikilink") 0\! は 1 と評価される。

  - 0 = 1 − 1 = 0 − 0、次は[4](https://ja.wikipedia.org/wiki/4 "wikilink")。()

  - [フィボナッチ数列](https://ja.wikipedia.org/wiki/フィボナッチ数列 "wikilink")では、0 は 1 の前者である。

  - [図形数](https://ja.wikipedia.org/wiki/図形数 "wikilink")では、1 の前に 0 を含めることがある。

  - 1桁は全て[回文数](../Page/回文数.md "wikilink")である。そのため、0 も回文数である。

### 数学におけるその他の用法

  - [集合論](../Page/集合論.md "wikilink")では  は空集合の[濃度である](../Page/濃度_\(数学\).md "wikilink")。ある人が林檎を一つも持っていないならば、その人は 0 個の林檎を持っている。実際のところ、集合論から展開されるある種の数学では 0 は空集合のこととして定義される。こう定義したとき、0 としての空集合は元を持たない集合としての空集合に対するであり、空集合に対する濃度は 0 個の元を持つという意味が割り当てられた値としての空集合を返す。
  - 同じく集合論で、 は最小の[順序数](../Page/順序数.md "wikilink")であり、空集合を[整列集合](https://ja.wikipedia.org/wiki/整列集合 "wikilink")とみなしたものに対応する。
  - [命題論理](../Page/命題論理.md "wikilink")では  を[真理値](https://ja.wikipedia.org/wiki/真理値 "wikilink")が[偽](https://ja.wikipedia.org/wiki/偽 "wikilink")であることを表すのに用いる。
  - [抽象代数学](../Page/抽象代数学.md "wikilink")では  は一般に（考えている構造において定義されているならば）加法に関する[単位元](../Page/単位元.md "wikilink")としての、あるいは乗法に関する[吸収元](https://ja.wikipedia.org/wiki/吸収元 "wikilink")としての、[零元](https://ja.wikipedia.org/wiki/零元 "wikilink")を表すのに用いられる。

  - 同じく抽象代数学において、零元のみからなる部分代数系  を  で表すことがある。（も参照）
  - [束論](https://ja.wikipedia.org/wiki/束論 "wikilink")では  は[有界束](https://ja.wikipedia.org/wiki/有界束 "wikilink")の[最大元](https://ja.wikipedia.org/wiki/最大元 "wikilink")を表すのに用いられる。
  - [圏論](../Page/圏論.md "wikilink")では  は[圏の](https://ja.wikipedia.org/wiki/圏_\(数学\) "wikilink")[始対象](https://ja.wikipedia.org/wiki/始対象 "wikilink")（特に[零対象](https://ja.wikipedia.org/wiki/零対象 "wikilink")）を表すのに用いられる。
  - [ゲーデル数](../Page/ゲーデル数.md "wikilink")では、 は[空文字列](../Page/空文字列.md "wikilink")を意味する。

## 自然科学における 0 の使用

### 物理学における使用

多くの[物理量](../Page/物理量.md "wikilink")において 0 は特別な値であるが、それは物理的な必然性を持って設定されることもあれば、何らかの任意の基準を適当に割り当てることもある。例えば[熱力学温度](../Page/熱力学温度.md "wikilink")における 0 度は理論的な最低温度（[絶対零度](../Page/絶対零度.md "wikilink")）である一方、[セルシウス度](../Page/セルシウス度.md "wikilink")の 0 度は（数ある物質の中から）[水](../Page/水.md "wikilink")の[融点](../Page/融点.md "wikilink")を選んで定義されている。

音の強さの単位である[デシベル](../Page/デシベル.md "wikilink")や[ホン](../Page/ホン.md "wikilink")は、基準として選んだ音の強さ（例えば、人間が聞き取れる最小の音量）を 0 と定めての相対値である。これは、倍率の表現に指数を使っているからである。すなわち「基準の1倍」が「基準の、(10のゼロ乗)倍」だからである（底としては10の他に2やeなどの場合もある）。

[零点振動](https://ja.wikipedia.org/wiki/零点振動 "wikilink")は[量子力学](https://ja.wikipedia.org/wiki/量子力学 "wikilink")（[不確定性原理](https://ja.wikipedia.org/wiki/不確定性原理 "wikilink")）において許される最低の[エネルギー](https://ja.wikipedia.org/wiki/エネルギー "wikilink")状態における原子の振動である。

## コンピュータにおける 0 の使用

初期の、[FORTRAN](../Page/FORTRAN.md "wikilink") や [COBOL](../Page/COBOL.md "wikilink") などのプログラミング言語では、配列の添字は 1 から始める方式であった（「1オリジン」という）。しかし、1950年代後半 の時点で既に [ALGOL](../Page/ALGOL.md "wikilink") 58 が柔軟な配列の添字（正、負、0 のいずれの整数も可）を導入している。現代の多くの言語は「0オリジン」であり、例えば[C言語](../Page/C言語.md "wikilink")では、n 個の要素を持つ配列の添字は 0 から n-1 までである。0オリジンには、配列の先頭アドレスに単に添字を足すだけで、その要素の位置を求めることが出来る利点がある。一方でBASICのように、古い標準（JIS C 6207-1982「基本BASIC」）では0オリジンだったのに、新しい標準（JIS X 3003:1993「Full BASIC」）では（変更可能だが）1オリジンがデフォルト、と逆行した例もある。

[ヌルポインタ](https://ja.wikipedia.org/wiki/ヌルポインタ "wikilink")はどんな[オブジェクト](https://ja.wikipedia.org/wiki/オブジェクト "wikilink")も指さない[ポインタである](../Page/ポインタ_\(プログラミング\).md "wikilink")。C言語においては整数定数の 0 がポインタの文脈で解釈されるとヌルポインタとなる。これは単なる記法であり、実際には計算機環境に適合した内部表現のヌルポインタが作られる（0番地と決まっているわけではない）。

0 はしばしばコンピュータにおいて特別な意味を持つ。C言語を始めとする多くの言語では、[真偽値](https://ja.wikipedia.org/wiki/真偽値 "wikilink")として評価する文脈において 0 は偽を意味すると判断される（0以外の全ての値は真と判断される）。一方、プログラムが戻り値として 0 を返した場合は正常終了と見なされる事が多い。errnoなどのエラーコードにおいても 0 は「エラーでない」の意味によく割り当てられる。[コードポイント](https://ja.wikipedia.org/wiki/コードポイント "wikilink")の 0 ('0')は[ヌル文字](https://ja.wikipedia.org/wiki/ヌル文字 "wikilink")であり、文字列の終端を意味する。

[−0](../Page/−0.md "wikilink") は、普通の算術では 0 とまったく同じものであるが、[1の補数](https://ja.wikipedia.org/wiki/1の補数 "wikilink")など、表現方法によっては別の表現が与えられることがある（現代の多くのコンピュータで採用されている[2の補数](../Page/2の補数.md "wikilink")では区別はない）。また、多くの[浮動小数点数](../Page/浮動小数点数.md "wikilink")では、-0.0 は別のものとして扱われることがある（詳細は「[IEEE 754における負のゼロ](https://ja.wikipedia.org/wiki/IEEE_754における負のゼロ "wikilink")」の記事を参照）。
