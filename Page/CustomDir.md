**CustomDir** （**カスタム・ディアー**）は、株式会社スプラインが提供していた、[Open Directory Projectの](https://ja.wikipedia.org/wiki/Open_Directory_Project "wikilink")[日本語](../Page/日本語.md "wikilink")階層[ウェブディレクトリ](../Page/ウェブディレクトリ.md "wikilink")のデータを改変した二次配布パッケージとウェブディレクトリ構築支援ツールを含む[フリーソフト](https://ja.wikipedia.org/wiki/フリーソフト "wikilink")の総称である。

Open Directory Projectは、[オープンコンテント](../Page/オープンコンテント.md "wikilink")であるため、そのデータ利用はライセンス下で自由度が高いものであったが、日本人が気軽に利用するには実際には多くの障壁があった。

  - [多言語ディレクトリ](../Page/多言語ディレクトリ.md "wikilink")であるために、多くの[文字コード](../Page/文字コード.md "wikilink")が混在するなどの煩わしさ
  - 全体のファイルサイズが、2GB以上であるため[サーバ](../Page/サーバ.md "wikilink")の記憶容量を占有してしまうこと
  - 契約文が[英語](../Page/英語.md "wikilink")で書かれているなどの語学的な難しさ
  - [ファイル作成のために](../Page/ファイル_\(コンピュータ\).md "wikilink")[プログラムを組まねばならないような技術的な難しさ](../Page/プログラム_\(コンピュータ\).md "wikilink")

などである。 それでも、[ドイツ語](../Page/ドイツ語.md "wikilink")など[ヨーロッパ](../Page/ヨーロッパ.md "wikilink")諸語でのデータ利用は活発であった。 日本語に関しては、2002年当時、わずかにAAA\! CAFEが利用するくらいであった（現在は利用していない）。二次配布サイトの[Thumbshot](https://ja.wikipedia.org/wiki/Thumbshot "wikilink")が登場した後も、多くの日本人にとって英語の壁はなお大きいものであった。 そして日本語化による再配布という発想でこの困難が解決された。

  - 多くの日本人にとって、英語などの大部分の外国語[ディレクトリ](../Page/ディレクトリ.md "wikilink")は関係がない。
  - 日本語階層だけに限定すれば、ファイルサイズを、100MB以下と小さくできる。
  - 英語の契約文を和訳する。
  - 解凍だけでファイルが作成される。

これをCustomDirが実践することで、日本語階層を利用する[ウェブサイト](../Page/ウェブサイト.md "wikilink")が増加しつつある。 ただし登録データは自動的に更新されるわけではないので、配布元のデータ更新を待つか設置者が手動で行なうしかないのが難点。 現在ではXOOPS用に生のデータをそのまま扱えるモジュール[Open Directory for XOOPSが配布されているので](https://ja.wikipedia.org/wiki/Open_Directory_for_XOOPS "wikilink")、 常時最新のデータを検索するにはこちらが便利だろう。
