ブロック () とは、[プログラミング言語](../Page/プログラミング言語.md "wikilink")におけるコードのまとまり（コードブロック）のことである。

## 概要

[文](https://ja.wikipedia.org/wiki/文_\(プログラミング\) "wikilink") (statement) から成る言語では、ブロックによって複数個（0/2個以上・言語により異なる）の文がまとまってひとつの文になっているものを複文 (compound statement) と呼ぶものもある。。。

自然言語の段落と異なり、ブロックは[入れ子にすることができる](../Page/ネスティング.md "wikilink")（）。すなわち、ブロック内にブロックを作成することができる（正確な理解としては、ブロックは文の一種であり、ブロックの中身は文の並びであるから、ブロックの中身にまたブロックが現れることもできる、ということになる。プログラミング言語の学習においては、絶対的に不正確な理解に直結する自然言語からの類推ではなく、プログラミング言語は[形式言語](../Page/形式言語.md "wikilink")なのであるから、[BNFなど形式的な](../Page/バッカス・ナウア記法.md "wikilink")（フォーマルな）仕様などを理解できるように努めたほうが、結果として早道である）。

[C言語](../Page/C言語.md "wikilink")などでは、ブロックは[変数の](../Page/変数_\(プログラミング\).md "wikilink")[スコープ](../Page/スコープ.md "wikilink")（可視範囲）の区切りである。すなわち、あるブロック内で定義された変数には、[ポインタ等によりエスケープされない限り](../Page/ポインタ_\(プログラミング\).md "wikilink")（[エスケープ解析](https://ja.wikipedia.org/wiki/エスケープ解析 "wikilink")を参照）、ブロック外からはアクセスできない。[JavaScript](../Page/JavaScript.md "wikilink") ([ECMAScript](../Page/ECMAScript.md "wikilink")) は、以前はブロックは変数のスコープに影響しないという仕様だったが、ECMAScript 6th Edition (ECMAScript 2015) で導入されたlet宣言 (let declaration) による変数は、それが宣言されたブロックの最後までが可視範囲となる。

自動変数（[ローカル変数](../Page/ローカル変数.md "wikilink")）の場合、ブロックによって変数の記憶域期間 (storage duration) が決まる\[1\]。一方、静的変数の記憶域期間は、ブロックによって左右されない。

[Smalltalk](../Page/Smalltalk.md "wikilink")や[Ruby](../Page/Ruby.md "wikilink")にある似たようなものは、ブロックと呼んではいるが、複文ではない別のもので、メソッド呼び出しに付加する特別な引数のようなもの、である。Rubyではそれ自体はオブジェクトではないが、Smalltalkではオブジェクトである。なお、Rubyにおける複文に相当するものとしては、if-endなどにおいて、ifそれ自体がコードのかたまりの開始のキーワードとして機能するという[Eiffel](../Page/Eiffel.md "wikilink")風のスタイルを採っている。

## ブロックの構文

ブロックの記述については、言語によって構文が異なる。

  - 括弧で囲む …… [C言語](../Page/C言語.md "wikilink")、[Java](https://ja.wikipedia.org/wiki/Java "wikilink")等
  - キーワードで囲む …… [Pascal](../Page/Pascal.md "wikilink")、[Visual Basic](../Page/Visual_Basic.md "wikilink")、[Ruby](../Page/Ruby.md "wikilink")、[Ada](../Page/Ada.md "wikilink")等
  - [インデント](https://ja.wikipedia.org/wiki/インデント "wikilink")で区別する ([オフサイドルール](../Page/オフサイドルール.md "wikilink")) …… [Python](../Page/Python.md "wikilink")等

[F\#のように](../Page/F_Sharp.md "wikilink")、軽量構文 (lightweight syntax) と冗語構文 (verbose syntax) を選択できる言語も存在する\[2\]。

## 例

入れ子になったブロックの例。

``` c
#include <stdio.h>

int main(void)
{
    int x = 0;

    while (1)
    {
        if (x < 10)
        {
            printf("%d\n", x);
            x++;
        }
        else
        {
            printf("Finished.\n");
            break;
        }
    }
    return 0;
}
```

## ブロッキング処理

動作が完了するまで呼び出し元が待たされる（ブロックされる）処理のことをブロッキング処理 (blocking processing) と呼ぶ。ブロックされる状況にある場合は即座にエラーを返して呼び出し元を待たせない方式をノンブロッキング処理 (non-blocking processing) と呼ぶ。代表的な例として、ネットワーク通信におけるブロッキングとノンブロッキングが挙げられる。

また、呼び出しが完了すると同時に処理結果を通知する方式を同期処理 (synchronous processing) と呼ぶ。いったん処理を開始してから、後で（[イベントや](../Page/イベント_\(プログラミング\).md "wikilink")[コールバック関数](https://ja.wikipedia.org/wiki/コールバック関数 "wikilink")を使って）結果を通知する方式を非同期処理 (asynchronous processing) と呼ぶ。通例ファイルなどのストレージ入出力処理 (Input/Output, I/O) には非同期I/Oを用いる。
