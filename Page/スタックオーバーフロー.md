**スタックオーバーフロー** () は、[コンピュータプログラムにおいて](../Page/プログラム_\(コンピュータ\).md "wikilink")、[コールスタック](https://ja.wikipedia.org/wiki/コールスタック "wikilink")領域の限界を超えたデータプッシュにより発生する、[バッファオーバーフローの一種である](https://ja.wikipedia.org/wiki/バッファオーバーラン "wikilink")。[スタックバッファオーバーフロー](https://ja.wikipedia.org/wiki/スタックバッファオーバーフロー "wikilink") () とは別の概念である。

## 概要

プログラムにおいて、[サブルーチン](../Page/サブルーチン.md "wikilink")（関数／プロシージャ）呼び出しに関する情報を格納するための[スタック](../Page/スタック.md "wikilink")メモリ領域（コールスタック）が確保される。サブルーチン呼び出しのたびにデータがスタックに積まれ（プッシュ）、サブルーチンが終わって[制御フロー](https://ja.wikipedia.org/wiki/制御フロー "wikilink")が呼び出し元に戻るとスタックからデータが降ろされる（ポップ）。[オペレーティングシステム](../Page/オペレーティングシステム.md "wikilink")や実行オプションにもよるが、コールスタックに格納できる情報量には上限がある。コールスタックに蓄積されるデータ量が限界を超えるとスタックは「オーバーフロー」し、未定義動作を引き起こす。[オペレーティングシステム](../Page/オペレーティングシステム.md "wikilink")にもよるが、プログラム側で対策をとっていなければ通常はプロセスがクラッシュしてしまう。ただし、スタックオーバーフローの原因となりうるコードパターンは比較的限定されている。

スタックオーバーフローの発生原因のうち、最も多いのはサブルーチンの[再帰呼び出し](https://ja.wikipedia.org/wiki/再帰呼び出し "wikilink")による[無限ループ](../Page/無限ループ.md "wikilink")（無限再帰）である。また、ソースコードの設計上は無限再帰ではなく、終了条件を決めて有限回数で打ち切るような有限再帰になっていたとしても、再帰呼び出しの階層数が深すぎると実行環境におけるコールスタックの上限を超えてしまい、スタックオーバーフローとなってしまう場合もある。ただし、[末尾最適化を実装した](../Page/末尾再帰.md "wikilink")[プログラミング言語](../Page/プログラミング言語.md "wikilink")では、末尾再帰の形で書かれたサブルーチンをループへ展開することができ、スタックオーバーフローは起こらなくなる。フラットなループ処理に変換されることで、再帰呼び出しによりスタックを消費することが無くなるからである。なお、再帰ではないサブルーチン呼び出しであっても、呼び出し階層数が深すぎる場合には、やはりスタックオーバーフローが起こりうる。

次によくある原因としては、スタック上に巨大な配列を確保しようとすることである。コールスタックに格納できる情報量には上限があるため、巨大なサイズの[ローカル変数](../Page/ローカル変数.md "wikilink")を定義するのではなく、[ヒープ領域](../Page/ヒープ領域.md "wikilink")などを明示的に利用すべきである。

## C/C++ での例

### 再帰による無限ループ

``` c
/* 関数の戻り値が int の場合、C言語では前方宣言や前方定義がなくてもコンパイル可能だが、C++では前方宣言あるいは前方定義が必要。 */
int g(void);

int f(void) {
    return g();
}

int g(void) {
    return f();
}
```

関数 `f()` は関数 `g()` を呼び出しているが、`g()` もやはり `f()` を呼び出している。交互の呼び出しは終わることがなく、最終的にはスタックオーバーフローが発生する。以下のような終了条件のない自己再帰も無限再帰となる。

``` c
int foo(void) {
    return foo();
}
```

### 巨大な配列をスタックに配置した場合

システムに実装されている[メインメモリ](https://ja.wikipedia.org/wiki/メインメモリ "wikilink")の容量にかかわらず、各プログラムのスタック領域のサイズは既定でせいぜい数[MiB程度](https://ja.wikipedia.org/wiki/メビバイト "wikilink")\[1\]\[2\]であり、大容量の[配列](../Page/配列.md "wikilink")を確保するのには向いていない。

``` c
#include <stdio.h>
int main(void) {
    int ary[1000 * 1000 * 1000]; /* 配列が大きすぎる */
    printf("Hello!\n"); /* &"Hello!\n" のプッシュがスタックオーバーフローとなる */
}
```

この場合は、

``` c
#include <stdio.h>
int ary[1000 * 1000 * 1000];
int main(void) {
    printf("Hello!\n");
}
```

または

``` c
#include <stdio.h>
int main(void) {
    static int ary[1000 * 1000 * 1000];
    printf("Hello!\n");
}
```

のように配列を[静的領域](https://ja.wikipedia.org/wiki/静的領域 "wikilink")に移動するか、あるいは [malloc](https://ja.wikipedia.org/wiki/malloc "wikilink") などを使って[ヒープ領域](../Page/ヒープ領域.md "wikilink")に動的確保すればスタックオーバーフローは回避できる。通例、システムが利用できる空きメモリの量は必ずしも定かではないので、実行時の[動的メモリ確保](../Page/動的メモリ確保.md "wikilink")と成否チェックを行なうことが望ましい。

``` c
#include <stdio.h>
#include <stdlib.h>
int main(void) {
    int *ary = (int *)malloc(sizeof(int) * 1000 * 1000 * 1000);
    printf("Hello!\n");
    free(ary);
}
```
