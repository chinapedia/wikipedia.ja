同図像性（どうずぞうせい、英：Homoiconicity）は、一部の[プログラミング言語](../Page/プログラミング言語.md "wikilink")が持つ特性である 。 ある言語のコードをその言語で操作できる場合、その言語は**同図像性がある**という。 「プログラムコードをデータとして扱う」と呼ばれることもある。

同図像性はデータモデルを表現するための言語とプログラム言語が深く関連しているか、同一であることを意味する。 同図像性は[メタプログラミング](../Page/メタプログラミング.md "wikilink")を容易にする。

同図像性を持つ代表的なプログラミング言語は[Lispである](https://ja.wikipedia.org/wiki/LISP "wikilink")。Lispの構造は[階層化されたリストの形式をとる](../Page/ネスティング.md "wikilink")[S式](https://ja.wikipedia.org/wiki/S式 "wikilink")によって与えられる。その結果、プログラムは実行中にある関数にアクセスでき、プログラムによってその場でその関数を変更できる。

### Lispの例

[Lispは](https://ja.wikipedia.org/wiki/LISP "wikilink")[S式](https://ja.wikipedia.org/wiki/S式 "wikilink")をデータとコードの外部[表現として使用する](https://ja.wikipedia.org/wiki/S式 "wikilink")。 S式は、プリミティブのLisp関数`READ`で読み取ることができる。 `READ`はLispデータを返す：リスト、 シンボル 、数値、文字列。 プリミティブのLisp関数`EVAL`は、Lispデータとして表されたLispコードを使用し、副作用を計算して結果を返す。 結果は、Lispデータから外部S式を作成するプリミティブ関数`PRINT`によって出力される。

``` lisp
((:name "john" :age 20) (:name "mary" :age 18) (:name "alice" :age 22))
```

Lispコード。 この例では、リスト、記号、数字を使用する。

``` lisp
(* (sin 1.1) (cos 2.03))   ; in infix: sin(1.1)*cos(2.03)
```

Lisp関数`LIST`を使用して上記の式を作成し、変数`EXPRESSION`を結果に格納する。

``` lisp
(setf expression (list '* (list 'sin 1.1) (list 'cos 2.03)) )
-> (* (SIN 1.1) (COS 2.03))  ; Lisp returns and prints the result

(third expression)  ; the third element of the expression
-> (COS 2.03)
```

`COS関数`を`SIN関数`に変更する。

``` lisp
(setf (first (third expression)) 'SIN)
; The expression is now (* (SIN 1.1) (SIN 2.03)).
```

式を評価する。

``` lisp
(eval expression)
-> 0.7988834
```

式を文字列に出力する。

``` lisp
(print-to-string expression)
-> "(* (SIN 1.1) (SIN 2.03))"
```

文字列から式を読み取る。

``` lisp
(read-from-string "(* (SIN 1.1) (SIN 2.03))")
-> (* (SIN 1.1) (SIN 2.03))   ; returns a list of lists, numbers and symbols
```
