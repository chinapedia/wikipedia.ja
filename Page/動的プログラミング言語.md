**動的プログラミング言語**（）は、コンピュータ・[プログラミング言語](../Page/プログラミング言語.md "wikilink")において、一般に[コンパイルなどの準備段階に済まされることが多いことを](../Page/コンパイラ.md "wikilink")、準備時ではなく実行時に行うプログラミング言語処理系（の言語）である。本来は実装（処理系）のことを指すべきであるが、一般にその言語を指して議論されることが多い。もっぱら[高水準言語](../Page/高水準言語.md "wikilink")が多い。**動的言語**（）とも。例えば、[型システム](../Page/型システム.md "wikilink")による型チェックや、値の[型変換](../Page/型変換.md "wikilink")、[名前束縛などを](https://ja.wikipedia.org/wiki/束縛_\(情報工学\)#名前束縛 "wikilink")、コンパイルなどの実行準備時ではなく実行時に行う。そういった「動的さ」により、[コードの追加や](../Page/ソースコード.md "wikilink")[オブジェクトや定義の拡張や](../Page/オブジェクト_\(プログラミング\).md "wikilink")[型システム](../Page/型システム.md "wikilink")の変更によるプログラムの拡張、といったことが行いやすい、ということが利点とされる。これらの動作は静的な処理系でも手間はかかるがエミュレート可能である（例えばJavaのJDK/JREでも、[クラスローダーを活用すれば可能である](https://ja.wikipedia.org/wiki/Javaクラスローダー "wikilink")）。一方、動的言語ではそういったことが、直接的にサポートされる。

動的言語と[動的型付け](../Page/動的型付け.md "wikilink")は同じ概念ではないし、全ての動的言語が必ずしも動的型付き言語というわけではない。たとえば[Smalltalk](../Page/Smalltalk.md "wikilink")環境の一つである[StrongtalkやGradualtalkは動的な言語にも拘わらず静的型検査が可能となっている](https://ja.wikipedia.org/wiki/:en_:Strongtalk "wikilink")。しかし多くの動的言語は動的型付き言語である。

## 定義の限界と曖昧さ

動的言語の定義は、「コンパイル」と「実行時」の区別だけでなく、「コード」と「データ」の区別も関わってくるため、非常にあいまいである。[仮想機械](../Page/仮想機械.md "wikilink")や[実行時コンパイラ](../Page/実行時コンパイラ.md "wikilink")など、[機械語](../Page/機械語.md "wikilink")に何らかの抽象化を施し、機械語を実行時に生成する言語処理系は多い。一般に、ある言語が動的であるとは、その言語の能力を明確に表しているというよりも、動的な機能の使い易さを指しているといえる。

## 実装

動的プログラミングの概念を密接な関連のある機構がいくつか存在する。それらは必ずしも動的言語として必須とは言えないが、動的言語と呼ばれるものに広く採用されているのも確かである。

###

とは、で導入された用語であり、[S式](https://ja.wikipedia.org/wiki/S式 "wikilink")と呼ばれるデータ構造で表現される命令列を実行するプロセスを指している。今日では、 と言えばテキストなど機械語以外のデータで表されたプログラムを実行する何らかの機構やプロセスを指す。多くの言語では（とは異なり）プログラムテキストを読むこととそれを内部表現に変換すること、さらには内部表現から実際の動作に変換することを区別しないため、新たなプログラムテキストの評価はそういった言語の一般的性質である。それらの言語で、プログラムの実行プロセスが  に他ならない場合、これを[インタプリタ](../Page/インタプリタ.md "wikilink")言語と呼ぶ。

#### 高階関数

とピーター・ドレイトンは、実行時に実行コードをロード可能な任意の言語は（たとえ、そのコードがDLLの機械語コードだとしても）、一種の  機能を備えていると言えるとした。彼らは[高階関数](../Page/高階関数.md "wikilink")が動的プログラミングの本質であるとし、「 は高階関数の代用品にすぎない」とした\[1\]。

### オブジェクトシステムや型システムの実行時変更

動的言語では、オブジェクトシステムや型システムを実行時に変更できるのが一般的である。これはすなわち、実行時定義に従って新たなオブジェクトを生成したり、既存の型やオブジェクトの合成に基づいて新たなオブジェクトを生成することを意味する。また、[継承関係や型の木構造の変更も指していて](../Page/継承_\(プログラミング\).md "wikilink")、既存の型の振る舞いも（特に[メソッド呼び出しの観点で](../Page/メソッド_\(計算機科学\).md "wikilink")）それによって変化する。[Smalltalk](../Page/Smalltalk.md "wikilink")のクラス登録などが顕著な例である。

``` smalltalk
Object
    subclass:               #Some
    instanceVariableNames:  ''
    classVariableNames:     ''
    poolDictionaries:       ''
    category:               'Example'.
```

上記はクラス定義用の専用構文ではなく、Objectというクラスオブジェクトにクラス登録用のメッセージを送っているだけである。実行中に登録しているが削除も実行中に可能である。

#### 制御構文

LispやSmalltalkをはじめ幾つかの言語は分岐や反復の専用の制御構文を持たない。無名関数と高階関数の組み合わせによって全てを代用している。

``` smalltalk
[
    Error signal: '処理失敗'.
]
    on: Exception
    do:
    [ :exception |
        "例外を捕捉"
    ].
```

上記はSmalltallkによる例外処理の例である。クラス登録と同じく言語機能による専用構文ではない。

### メッセージ送信

Smalltalk系統のオブジェクト指向言語にみられる。一つのメソッドに複数セレクターと紐づけたり、全てのメッセージを受け取るメソッドを定義することで、メッセージを受信するオブジェクトがメッセージと同じセレクターを持つメソッドを備えていなくてもメッセージを受信できる。

``` smalltalk
"#onClick:を使ったメッセージをopen:メソッドに転送させる。"
FileEventHandler
    addSelector:    #onClick:
    withMethod:     FileEventHandler >> #open:.
```

上記はSmalltalkで既に\#open:というセレクターを持っているメソッドに\#onClick:というセレクターを追加している。

「[メッセージ転送](https://ja.wikipedia.org/wiki/メッセージ転送 "wikilink")」も参照のこと。

### 関数型プログラミング

[関数型プログラミングの概念は多くの動的言語に備わっている](../Page/関数型言語.md "wikilink")。これも LISP を起源としている。

#### 継続

動的言語における別の機能として[継続](../Page/継続.md "wikilink")がある。継続は、再呼び出し可能な実行状態を表している。例えば、[構文解析器](../Page/構文解析器.md "wikilink")が中間結果と継続を返し、後で再呼び出しされると、入力の構文解析を再開する。継続はクロージャなどのスコープと複雑に関連している。このため、継続を提供している動的言語は多くはない。

### イントロスペクション

[イントロスペクション](https://ja.wikipedia.org/wiki/イントロスペクション "wikilink")は多くの動的言語で見られ、自分自身の構造・コード・型・データなどを解析するプログラムによく使われる。これは[総称型](https://ja.wikipedia.org/wiki/総称型 "wikilink")や[ポリモーフィズム](../Page/ポリモーフィズム.md "wikilink")の値について型を決定するのと似たようなものである。プログラムのコードをデータとして完全に解析可能な場合もあり、例えば LISP ではプログラムをS式として解析可能である。

### マクロ

動的プログラミング言語の中には、コードのイントロスペクションと eval 機能を組み合わせて[マクロと呼ばれる機能を提供するものもある](../Page/マクロ_\(コンピュータ用語\).md "wikilink")。[C言語](../Page/C言語.md "wikilink")や[C++](../Page/C++.md "wikilink")におけるマクロは、プログラム内の文字列を置換するサブセット的かつ静的な機能でしかない。これに対し動的言語では、マクロはコンパイラ内部へのアクセスを提供し、実行時にも完全なアクセスを提供する。従って、コードの最適化にも関わり、言語の文法や構文を変化させる能力を持っている。
