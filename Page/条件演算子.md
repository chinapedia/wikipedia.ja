**条件演算子**（）は、[プログラミング言語](../Page/プログラミング言語.md "wikilink")の[演算子](../Page/演算子.md "wikilink")で、[条件文と同様な意味があるが](../Page/If文.md "wikilink")、[文ではなく](https://ja.wikipedia.org/wiki/文_\(プログラミング\) "wikilink")[値を持つ](../Page/値_\(情報工学\).md "wikilink")[式になる](../Page/式_\(プログラミング\).md "wikilink")。評価されると、条件式の値により異なる式が評価され、異なる値になる。

なお、[C言語](../Page/C言語.md "wikilink")や[C++](../Page/C++.md "wikilink")など一部のプログラミング言語において、条件演算子とは後述する「? :」([ハテナマーク](https://ja.wikipedia.org/wiki/ハテナマーク "wikilink")と[コロン](../Page/コロン_\(記号\).md "wikilink")) の演算子の名称である。

## 概要

「If {{～}} Then {{～}} Else {{～}}」にあたる[三項演算子](https://ja.wikipedia.org/wiki/三項演算子 "wikilink")があり、ほぼ唯一の、プログラミング言語において一般的な三項演算子である。また「If {{～}} Then {{～}} if」にあたる[二項演算子](https://ja.wikipedia.org/wiki/二項演算子 "wikilink")もある。

三項演算子は、条件式・真式・偽式の3つの[オペランド](https://ja.wikipedia.org/wiki/オペランド "wikilink")を結ぶ。二項演算子の場合は、条件式と、真式または偽式のいずれかの、2つのオペランドを結ぶ。

条件文との違いは、

  - 真式・偽式は式（条件文における節のように文を置くことができない）
  - 式であるため値を持つ

である。

## 種類

### 三項演算子

[C言語](../Page/C言語.md "wikilink")などには、次の構文の三項の条件演算子がある。

<条件式>` ? `<真式>` : `<偽式>

3項を結ぶ必要上、「[?](../Page/疑問符.md "wikilink")」と「[:](../Page/コロン.md "wikilink")」の2つの記号を使うが、演算子としては1つの演算子である。

<条件式>は[真](https://ja.wikipedia.org/wiki/真 "wikilink") (true) または[偽](https://ja.wikipedia.org/wiki/偽 "wikilink") (false) の[論理値](https://ja.wikipedia.org/wiki/論理値 "wikilink")を返す[論理式](https://ja.wikipedia.org/wiki/論理式 "wikilink") ([ブール型](https://ja.wikipedia.org/wiki/ブール型 "wikilink")) である（あるいは論理値に暗黙に[型変換](../Page/型変換.md "wikilink")可能な[型を返す](../Page/データ型.md "wikilink")）。真式と偽式は同じ型の値を返さなければならない（暗黙に型変換可能なら許されることもある）。

この構文自身も[式であるので値をもち](../Page/式_\(プログラミング\).md "wikilink")、その値は上記<真式>または<偽式>の値のいずれかである。<条件式>が真の場合は<真式>、偽の場合は<偽式>の値となる。

例えば、[Java](https://ja.wikipedia.org/wiki/Java "wikilink")言語では、次のように用いる。

``` java
int weight;
...
String message = weight <= 100 ? "OK" : "積載量オーバー";
```

この場合、変数`weight`の値が`100`以下であれば変数`message`に`"OK"`という文字列が代入され、それ以外であれば`"積載量オーバー"`という文字列が代入される。これは通常の[if-else文を用いて書いた次の例と同じ意味であり](https://ja.wikipedia.org/wiki/if文 "wikilink")、[コンパイル](https://ja.wikipedia.org/wiki/コンパイル "wikilink")した結果はしばしば同じものとなる。

``` java
String message;
if (weight <= 100) {
    message = "OK";
} else {
    message = "積載量オーバー";
}
```

条件演算子はこの例のように、条件によって異なる値をある処理（上の例では`message`への代入）に適用する、という場合に記述を簡潔にすることができる。

[C言語](../Page/C言語.md "wikilink")および[C++](../Page/C++.md "wikilink")では、変数宣言時の初期化に条件演算子を用いることで、再代入を許可しない`const`修飾された変数を定義することができるようになる。さらにC++においては、if-else文による初期化では必要な、変数宣言時のデフォルト[コンストラクタ](../Page/コンストラクタ.md "wikilink")が不要となる。

``` c++
const std::string message(weight <= 100 ? "OK" : "積載量オーバー");
```

#### 演算子の優先順位

Cや[Java](https://ja.wikipedia.org/wiki/Java "wikilink")や[Perl](../Page/Perl.md "wikilink")などでは、`a ? b : c ? d : e` は `a ? b : (c ? d : e)` という意味だが、[PHPの三項演算子](../Page/PHP_\(プログラミング言語\).md "wikilink")\[[http://www.php.net/manual/en/language.operators.comparison.php\#language.operators.comparison.ternary\]では](http://www.php.net/manual/en/language.operators.comparison.php#language.operators.comparison.ternary%5Dでは) `(a ? b : c) ? d : e` なので注意が必要である。

### 論理演算子

二項の論理演算子（Cの標準規格の用語では「論理AND演算子」「論理OR演算子」）も、条件演算子のような意味を持っており（[短絡評価](../Page/短絡評価.md "wikilink")）、[Java](https://ja.wikipedia.org/wiki/Java "wikilink")では条件AND演算子と条件OR演算子という名前である。

<条件式>` && `<真式>
<条件式>` || `<偽式>

1行目は、<条件式>が真の場合にのみ<真式>が評価される。偽の場合は条件式が評価されるのみである。式の値は、<条件式>が真の場合は<真式>、偽の場合は<条件式>がそのまま返される。

2行目の構文も、真偽が逆であることを除けば同様である。

なお、Javaの演算子 `&` および `|` はオペランドの型が`boolean`型の場合は[ビット演算子](https://ja.wikipedia.org/wiki/ビット演算子 "wikilink")ではなく論理演算子となるが、両方のオペランドを必ず評価し、短絡評価はされない。[C\#も同様である](../Page/C_Sharp.md "wikilink")。

[Pascal](../Page/Pascal.md "wikilink")の`and`演算子と`or`演算子が短絡評価かどうかは処理系依存である。

### Ruby

[Ruby](../Page/Ruby.md "wikilink")では、C と同じ形で三項演算子を使うことができる。しかし、Ruby の場合は if も文ではなく式なので、`( A ? B : C )` ではなく `if A then B else C end` と書くほうが好まれている。

### Python

[Python](../Page/Python.md "wikilink")においては、Cの条件演算子と同じ機能は、構文に関する論争のため長い間実装されなかったが（論理演算子を使ったハックでなんとか似たことができる\[1\]というのもあった）、PEP 308として承認され、2006年9月の2.5 releaseに追加された\[2\]。

Python ではこの機能の式を条件式と呼び、構文は、C言語などとは順序が異なっている（新しい演算子ないしキーワードは導入されなかった）。[Perl](../Page/Perl.md "wikilink")の後置if修飾子のような見た目になった。

``` python
<真の場合の値> if <条件式> else <偽の場合の値>
```

### Visual Basic

[Visual Basic](../Page/Visual_Basic.md "wikilink")（2008より前）、[VBAには演算子はないが](../Page/Visual_Basic_for_Applications.md "wikilink")、同じように使える[関数](https://ja.wikipedia.org/wiki/関数_\(プログラミング\) "wikilink") Iif がある。

`Iif(`<条件式>`, `<真式>`, `<偽式>`)`

ただし条件演算子と違い、真式・偽式いずれも関数の引数なので、条件式の真偽にかかわらず双方とも評価される。条件に合わない側の値は捨てられるので値に影響はないが、真式・偽式に[副作用がある](../Page/副作用_\(プログラム\).md "wikilink")（何か出力する、[グローバル変数](../Page/グローバル変数.md "wikilink")を書き換えるなど）場合は挙動が異なってくる。

Visual Basic 2008以降には\[3\]、条件演算子と同様の短絡評価をおこなうIf演算子がある。

`If(`<条件式>`, `<真式>`, `<偽式>`)`
`'または`
`If(`<式1>`, `<式2>`)`

3引数の場合は、if-then-elseと同様の意味である。2引数の場合は、式1がNothingでない場合は式1の値を、式1がNothingの場合は式2の値を返す\[4\]。

## 条件演算子とif式

条件演算子をサポートしない言語であっても、ifが文ではなく式となっている言語では、if式を用いることで同等の機能を実現できる。例えば[F\#には組み込みの条件演算子はないが](../Page/F_Sharp.md "wikilink")、以下のようにif式で代用できる。

``` fsharp
printfn "Now %s." (if (System.DateTime.Now.Hour >= 12) then "PM" else "AM")
```

その他、無名関数を応用したものもある。例えばSmalltalkでは関数呼び出しに相当するメッセージ式に無名関数に相当するブロックを渡すことで条件式を実現している。

``` smalltalk
value := 0 < 1 ifTrue:[ 1 ] ifFalse: [ 2 ].
```

なお、この\#ifTrue:\#ifFalse:は分岐専用の構文ではなくメッセージ式の一例に過ぎない。メッセージ式は誰でも追加できるため、標準のライブラリーだけでも下記のような亜種が存在する。

``` smalltalk
value := 0 < 1 ifTrue:[ 1 ].
value := value ifNil:[ 1 ] ifNotNil[ 2 ].
value := value ifNil:[ 1 ].
value := value ifNotNil:[ :argument | 1 ].
value := value at: 1 ifAbsent:[ 0 ].
```

## 関連する演算子

  - 条件演算子: `A ? B : C`
  - [Null合体演算子](https://ja.wikipedia.org/wiki/Null合体演算子 "wikilink"): `A ?? B`など (null時のデフォルト値設定)
  - [エルビス演算子](https://ja.wikipedia.org/wiki/エルビス演算子 "wikilink"): `A ?: B` (false時のデフォルト値設定)
  - [Null条件演算子](https://ja.wikipedia.org/wiki/Null条件演算子 "wikilink"): `A?.B`など (nullチェックの自動化)
