**2元対称通信路**（[英](../Page/英語.md "wikilink"): **Binary symmetric channel**、**BSC**）とは、[符号理論](../Page/符号理論.md "wikilink")や[情報理論](../Page/情報理論.md "wikilink")でよく使われる[通信路](https://ja.wikipedia.org/wiki/通信路 "wikilink")モデルである。このモデルでは、送信者が1つの[ビット](../Page/ビット.md "wikilink")（0 か 1）を送信しようとし、受信者は1つのビットを受信しようとする。ビットは通常は正しく転送されるが、ある小さな[確率](https://ja.wikipedia.org/wiki/確率 "wikilink")（crossover probability）で反転したビットが受信されることがある。解析が最も容易な通信路であることから、情報理論で頻繁に使われる。

## 概要

[600px](https://ja.wikipedia.org/wiki/ファイル:Binary_symmetric_channel_\(en\).svg "wikilink") BSC は「2元通信路」である。つまり、2つの記号（一般に 0 と 1 とされる）のどちらかしか転送できない。非2元通信路は2種類以上の記号を転送可能である。その転送は完全ではなく、受信者は時折間違ったビットを受信してしまう。

この通信路は、[ノイズ](../Page/ノイズ.md "wikilink")のある通信路としては最も解析が容易であるため、理論研究でよく使われる。における様々な問題は BSC に[還元できる](../Page/還元_\(計算複雑性理論\).md "wikilink")。一方、BSC での効率的な転送が可能なら、もっと複雑な通信路にその方法を応用することができる。

## 定義

crossover probability（ビットが反転する確率）が *p* の2元対称通信路はバイナリ入力とバイナリ出力と誤り確率 *p* の通信路からなるとする。*X* を送信する[確率変数](https://ja.wikipedia.org/wiki/確率変数 "wikilink")、*Y* を受信する変数としたとき、この通信路の特性は次のような[条件付き確率](https://ja.wikipedia.org/wiki/条件付き確率 "wikilink")で表される。

  -
    Pr( *Y* = 0 | *X* = 0) = 1-*p*
    Pr( *Y* = 0 | *X* = 1) = *p*
    Pr( *Y* = 1 | *X* = 0 ) = *p*
    Pr( *Y* = 1 | *X* = 1 ) = 1-*p*

ここでは、0 ≤ *p* ≤ 1/2 であると仮定している。*p*\>1/2 だった場合は、受信者が受信結果を反転（1 だったら 0、0 だったら 1 とみなす）させれば、1-*p* ≤ 1/2 となる。

### BSC の通信路容量

[通信路容量](../Page/通信路容量.md "wikilink")は 1 - H(*p*) であり、H(*p*) は[2値エントロピー関数](https://ja.wikipedia.org/wiki/2値エントロピー関数 "wikilink")である。

[球充填](https://ja.wikipedia.org/wiki/球充填 "wikilink")の考え方により以下が示される。符号語を与えられたとき、[典型集合](https://ja.wikipedia.org/wiki/典型集合 "wikilink")の出力ビット列はおよそ 2 <sup>*n* H(*p*)</sup> となる。考えられる出力は全部で 2<sup>*n*</sup> であり、入力は 2<sup>*nR*</sup> の符号語から選ばれる。従って、受信者は考えられる出力それぞれについて、2<sup>*n*</sup> / 2<sup>*nR*</sup> = 2<sup>*n*(1-*R*)</sup> の球で空間の分割を決定できる。*R*\> 1 - H(*p*) だった場合、球は詰め込みすぎとなって、受信者は出力から正しい符号語を特定できなくなる。
