[lang=ja](https://ja.wikipedia.org/wiki/ファイル:Euler_diag_for_jp_charsets.svg "wikilink")（JIS X 0208、JIS X 0212、JIS X 0213等の漢字集合）\]\] **JIS X 0213**は[JIS X 0208](../Page/JIS_X_0208.md "wikilink"):1997を拡張した[日本語](../Page/日本語.md "wikilink")用の[符号化文字集合](https://ja.wikipedia.org/wiki/符号化文字集合 "wikilink")をななは規定する[日本産業規格](../Page/日本産業規格.md "wikilink") (JIS) である。規格名称は「**7ビット及び8ビットの2バイト情報交換符号化拡張漢字集合**」である\[1\]。

2000年に制定、2004年、2012年\[2\]に改正された。2000年に制定されたJIS X 0213:2000は通称「**JIS2000**」と呼ばれている。2004年に改されたJIS X 0213:2004は通称「**JIS2004**」と呼ばれている。

JIS X 0208を拡張した規格で、JIS X 0208が規定する6879字の[図形文字](https://ja.wikipedia.org/wiki/図形文字 "wikilink")の集合に対して、日本語の文字コードで運用する必要性の高い4354字が追加され、計1万1233字の図形文字を規定する。JIS X 0208を拡張する点において[JIS X 0212](../Page/JIS_X_0212.md "wikilink"):1990と同目的であるが、JIS X 0212とJIS X 0213との間に[互換性](../Page/互換性.md "wikilink")はない。JIS X 0212がJIS X 0208にない文字を集めた文字集合であるのに対し、JIS X 0213はJIS X 0208を包含し更に第三・第四水準漢字などを加えた[上位集合](https://ja.wikipedia.org/wiki/上位集合 "wikilink")である。

## 特徴

[JIS X 0212](../Page/JIS_X_0212.md "wikilink")（補助漢字）が頻度調査を中心に追加文字を選定し、典拠用例などは[諸橋大漢和への参照情報を付加した程度だったのに対し](../Page/大漢和辞典.md "wikilink")、JIS X 0213ではJIS X 0208:1997 (97JIS) においてJIS X 0208の収載字体の用例・典拠を徹底して調べ上げ、同定したのと同様の手法で一般に使われる字（狭義の[字体](../Page/字体.md "wikilink")。以後「字」は狭義の「字体」を指す ）でJIS X 0208に収録されていないものを追加した。そのため、JIS X 0212と同じ字が含まれていることもあるが、JIS X 0212では収録されていてもJIS X 0213では[包摂規準を使って特に増やさなかった字がある](../Page/包摂_\(文字コード\).md "wikilink")。

拡張にあたっては、JIS X 0208の1997年改正で保留領域とされた部分に字を増やす方針で行われ、非漢字659字、第三水準として1249字、第四水準として2436字を追加した。実装では、JIS X 0208:1997で保留領域とされた部分に非漢字及び第三水準の文字を入れて第一[面とし](https://ja.wikipedia.org/wiki/符号面 "wikilink")、その後ろに第四水準の文字を第二面として加えた。さらに2004年の改正で、第三水準に10字が追加され、168字の例示字形が変更された。

第二面は第一面と同じく94区94点で構成されているが、そのうち文字の存在する符号領域は1, 3–5, 8, 12–15, 78–94区に限られる。この奇妙な配置は[JIS X 0212補助漢字の存在する場所を避けた結果である](../Page/JIS_X_0212.md "wikilink")。これにより[EUCエンコードされた文章でJIS](../Page/Extended_Unix_Code.md "wikilink") X 0212補助漢字を用いたものとJIS X 0213第二面を用いたものの判別が可能である。さらに両方を用いることも原理的には可能である（ただし、その規格は存在しない）。

しかし、JIS X 0208:1997で保留領域とされた部分は、過去のJIS X 0208で自由領域とされ、実装各社によって外字領域として使用されていた部分であり、実態としては既に使われている領域であった。ここに新たに文字を配置した規格案に対し実装各社側より意見があり、最終審査において各種符号化方式が「参考」（規格本体ではない）とされることになった。その一方で、JIS X 0208の空き領域を規定通り未使用としていた[UNIX](../Page/UNIX.md "wikilink")系ソフトウェアでは対応が比較的容易であり、複数の実装が存在する。

## 文字の表記方法

JIS X 0213ではJIS X 0208まで用いられていた「区点」に「[面](../Page/面_\(文字コード\).md "wikilink")」を加え「面区点」となり、「面-区-点」でコード表記を行う。例えば1面3区33点の「A」は「1-3-33」とあらわす。

## 符号化方式

### JISベースの文字コード

[符号化方式は](https://ja.wikipedia.org/wiki/文字符号化方式 "wikilink")、[ISO/IEC 2022にそった形のみ](https://ja.wikipedia.org/wiki/ISO/IEC_2022 "wikilink")「規定」としてあり、[ISO-2022-JP-2004](https://ja.wikipedia.org/wiki/ISO-2022-JP-2004 "wikilink")、[Shift_JIS-2004](../Page/Shift_JIS-2004.md "wikilink")、[EUC-JIS-2004](../Page/EUC-JIS-2004.md "wikilink")は「参考」として記述がある。これらのコード名は今のところ[IANAが登録していないので](../Page/Internet_Assigned_Numbers_Authority.md "wikilink")、[MIME等では](https://ja.wikipedia.org/wiki/Multipurpose_Internet_Mail_Extension "wikilink") "X-" で始まる私用の名称として用いる必要があることになる。 Shift_JIS-2004は、[macOS](https://ja.wikipedia.org/wiki/macOS "wikilink")や[Java 7などでは既に実装しているが](https://ja.wikipedia.org/wiki/Java "wikilink")、Windowsでは従来のシフトJIS（[コードページ932](../Page/Microsoftコードページ932.md "wikilink")）と互換性がないことを理由に実装していないため、広く利用することができない。

### Unicodeとの対応

JIS X 0213制定当時はいくつかの文字に対応する[Unicode](../Page/Unicode.md "wikilink")符号が存在しなかったが、Unicode 3.1およびUnicode 3.2で追加された。ただし、符号化にあたり注意点がある。

#### サロゲートペア

  - 漢字の内[CJK互換漢字](../Page/CJK互換漢字.md "wikilink")領域に追加されたものを除くと[基本多言語面](../Page/基本多言語面.md "wikilink") (BMP) 外のCJK統合漢字 拡張B領域に追加されることとなった。該当する文字は初版に302字、2004年追加分に1字の計303字ある。これらを使用する場合は、[UTF-8](../Page/UTF-8.md "wikilink")では4バイト長コードに、[UTF-16](../Page/UTF-16.md "wikilink")では[サロゲートペア（代用対）に対応する必要がある](https://ja.wikipedia.org/wiki/Unicode#サロゲートペア "wikilink")。[UTF-32](../Page/UTF-32.md "wikilink")に対応している場合はそのまま使用可能である。
      - 例えば1面14区2点の点の付いた「」はU+2000Bに割り当てられた。

#### 合成文字

  - 非漢字の内半濁点付き仮名、アクセント付きIPA記号で従来のUnicodeに単独の符号として無いもの、[声調の上昇調、下降調を示す記号は](https://ja.wikipedia.org/wiki/声調記号 "wikilink")2つのUnicode符号を組み合わせて表すこととなった。該当する文字は全部で25字ある。これらを[OpenType](../Page/OpenType.md "wikilink")で使用するには、オペレーティングシステムやアプリケーションが、OpenTypeのグリフ置換機能に対応する必要がある（この場合グリフ置換のうち、複数の隣り合うグリフをある一つのグリフに置換する機能を使用）。
      - 例えば1面4区87点の[半濁点](../Page/半濁点.md "wikilink")付き「」は「か」のU+304Bの後に合成用半濁点のU+309Aを付けて表すこととなった。
      - 1面11区69点の声調記号上昇調および1面11区70点声調記号下降調はU+02E5とU+02E9の組み合わせで表されるが、これはUnicode BookのChapter 7.8に基づくものである。これによると、U+02E5-U+02E9の5つの記号のうち複数が隣り合うと、上下の声調変化を示す記号ができるというものである。

#### CJK互換漢字の正規化

  - JIS X 0213とUnicodeでは[包摂規準が異なる](../Page/包摂_\(文字コード\).md "wikilink")。そこで JIS X 0213 での人名用漢字の字形（字体）を区別するために、一部の文字をUnicodeでは[CJK互換漢字](../Page/CJK互換漢字.md "wikilink")として収録している。CJK互換漢字は、[Unicode正規化](../Page/Unicode正規化.md "wikilink")によりCJK統合漢字に分解（変換）される。この対応として互換漢字用の[異体字セレクタ](../Page/異体字セレクタ.md "wikilink")（SVS）を使用して変換前の情報を維持する必要がある。また、CJK統合漢字との組み合わせを1文字として処理する必要もある。SVS対応フォントとしては[IPAexフォントなどがある](https://ja.wikipedia.org/wiki/IPAフォント#IPAexフォント "wikilink")。
      - 例えば、[KS X 1001由来のCJK互換漢字が誤って入力された場合に](../Page/KS_X_1001.md "wikilink")、入力エラーとせずUnicode正規化処理で対処する方式（CJK統合漢字のみに変換）を採用すると、CJK互換漢字に収録されている人名用漢字が入力された場合に意図せず変換されてしまう。\[4\]\[5\]
      - 例えば、<span style="font-family:IPAex明朝"></span>（U+FA19）をUnicode正規化すると<span style="font-family:IPAex明朝"></span>（U+795E）となる。これをSVSを使って <span style="font-family:IPAex明朝"></span>（U+795E U+FE00）とすることで元のCJK互換漢字（の情報、字形）を維持する（[FireFoxなど一部のブラウザでしかSVSの](../Page/Mozilla_Firefox.md "wikilink")[数値文字参照](https://ja.wikipedia.org/wiki/数値文字参照 "wikilink")に対応していないため、ブラウザにより表示が異なることに注意が必要\[6\]）。

#### CJK互換漢字のIVS対応

  - CJK互換漢字やCJK統合漢字+SVSを使用せず、CJK統合漢字との組み合わせ（IVS/IVD対応）により異体字（Unicodeでは包摂されるが、JIS X 0213として別の字体）を表現する場合、JIS X 0213の文字をこの組み合わせに変換する方法（複数あるため1つの方法を選んで変換）と、組み合わせを処理および表現（IVS対応フォントの導入）する必要がある。[異体字セレクタ\#実装](https://ja.wikipedia.org/wiki/異体字セレクタ#実装 "wikilink")も参照。

:\* 例えば、IVSとして[Adobe-Japan1](../Page/Adobe-Japan1.md "wikilink")コレクションを使用すると<span style="font-family:'Hiragino Mincho ProN','YuMincho','游明朝'"></span>（U+FA19）は<span style="font-family:'Hiragino Mincho ProN','YuMincho','游明朝'"></span>（U+795E U+E0100）となる。（[游書体](https://ja.wikipedia.org/wiki/游書体 "wikilink")、[ヒラギノ](../Page/ヒラギノ.md "wikilink")、[花園フォント](https://ja.wikipedia.org/wiki/花園フォント "wikilink")などのIVS対応フォントが必要である）

:\* 例えば、IVSとして文字情報基盤整備事業のMoji_Johoコレクションを使用すると<span style="font-family:HanaMinA"></span>（U+FA19）は<span style="font-family:HanaMinA"></span>（U+795E U+E0103）となる。（[IPAmj明朝](https://ja.wikipedia.org/wiki/IPAフォント#IPAmj明朝 "wikilink")、[花園フォント](https://ja.wikipedia.org/wiki/花園フォント "wikilink")などのIVS対応フォントが必要である）

#### 各OSでの対応状況

なお、[Windows Vista以降や](https://ja.wikipedia.org/wiki/Microsoft_Windows_Vista "wikilink")[macOS](https://ja.wikipedia.org/wiki/macOS "wikilink")ではこれらに対応している。[Windows XPではサロゲートペア](../Page/Microsoft_Windows_XP.md "wikilink")（代用対）に対応しており、Service Pack 2以上を適用することによってグリフ置換にも対応する。[Windows 2000はサロゲートペアに対応しているものの初期設定では無効化されておりレジストリの設定が必要である](../Page/Microsoft_Windows_2000.md "wikilink")（[Help:特殊文字\#古代文字と人工文字参照](https://ja.wikipedia.org/wiki/Help:特殊文字#古代文字と人工文字 "wikilink")）。またグリフ置換には未対応である。

アプリケーション側の対応も必要である。[Microsoft OfficeのXP以降のバージョンやWindows](../Page/Microsoft_Office.md "wikilink") Vistaに付属する[Internet Explorer](../Page/Internet_Explorer.md "wikilink") 7.0、Windows XP以降に付属するメモ帳やワードパッドなどでは対応済みである。

### ほかの実装方法

  - JIS X 0213が制定されてすぐのころは、UnicodeにはJIS X 0213が実装されていなかったため、[外字](../Page/外字.md "wikilink")領域に文字を定義して使用することが多く行われていた。いくつかの[フリーフォントではそのような実装が行われている](../Page/フリーソフトウェア.md "wikilink")\[7\]。
  - [日本のデータ放送](https://ja.wikipedia.org/wiki/日本のデータ放送 "wikilink")などで使用される[ARIB STD-B24では](https://ja.wikipedia.org/wiki/ARIB_STD-B24 "wikilink")、[ARIBの文字符号化方法の文字セットとしてJIS](https://ja.wikipedia.org/wiki/ARIB外字 "wikilink") X 0213が使用可能となっている。「JIS互換漢字1面集合」でJIS X 0213:2004の1面、「JIS互換漢字2面集合」でJIS X 0213:2004の2面が使用される。この符号化方式の中には「国際符号化文字集合」を使用する方法があり、通常のUnicodeのマッピングのほか、「BMPセット」としてUnicodeでの「基本多言語面」で全てを表現できるようにJIS X 0213での「[追加漢字面](https://ja.wikipedia.org/wiki/追加漢字面 "wikilink")」の文字を「[基本多言語面](../Page/基本多言語面.md "wikilink")」の[外字](../Page/外字.md "wikilink")領域にマッピングしなおした文字セットが別に使用可能とされている。
  - JAVA 7では通常のShift_JIS-2004形式の実装(x-SJIS_0213)のほかShift_JIS-2004に[Microsoftコードページ932](../Page/Microsoftコードページ932.md "wikilink")を上書きした符号化方式(x-MS932_0213)に対応している。

## 先行国内実装との互換性

JIS X 0213の第1面13区には、[PC-9801](https://ja.wikipedia.org/wiki/PC-9801 "wikilink")から続く[機種依存文字](../Page/機種依存文字.md "wikilink")とされてきた[NEC特殊文字が](https://ja.wikipedia.org/wiki/Microsoftコードページ932#NEC_特殊文字・IBM_拡張文字 "wikilink")、一部を除き同じ面区点番号で登録されている。

このため、これまで典型的な[機種依存文字](../Page/機種依存文字.md "wikilink")として挙げられてきたNEC特殊文字を使った文書がShift_JIS-2004として解釈できるようになった。

### 例示字形の変更

JIS X 0213:2004は、JIS X 0213:2000の例示字形を変更している。変更があったのは表1の通りである。

全部で168字あり、おおむね、[拡張新字体](../Page/拡張新字体.md "wikilink")から、いわゆる[康熙字典](../Page/康熙字典.md "wikilink")体型へ変更されている。「叉」や「釜」などは筆押さえを取っている。また、「蟹」（「解」と「虫」を離した）や「祟」（出を小さくした）などのようにもある。

この変更は、[国語審議会](../Page/国語審議会.md "wikilink")の答申「[表外漢字字体表](https://ja.wikipedia.org/wiki/表外漢字字体表 "wikilink")」に示されている字体に合わせるものである。ただし、文字コード規格とは文字の符号化表現を定めるものであり、例示字形は何ら規範的な役割を持たないことを謳っていながら、こうした変更を行うことに対して批判的な意見が公開レビューなどで寄せられた。 {{-}}

### フォントの対応

  - [マイクロソフト](../Page/マイクロソフト.md "wikilink")は[2007年](../Page/2007年.md "wikilink")に発売した[Windows Vistaから](https://ja.wikipedia.org/wiki/Microsoft_Windows_Vista "wikilink")、標準搭載フォント\[8\]の字形を旧来のJIS X 0208:1990のものから、JIS X 0213:2004のものに変更した。また、旧来の字形を表示できるフォントパッケージも公開された\[9\]。
  - [アップルも](../Page/アップル_\(企業\).md "wikilink")2007年発売の[Mac OS X v10.5において](../Page/Mac_OS_X_v10.5.md "wikilink")、例示字形の変更に対応した[ヒラギノ Nフォントを追加している](../Page/ヒラギノ.md "wikilink")。
  - [IPAが](../Page/情報処理推進機構.md "wikilink")[Linux](../Page/Linux.md "wikilink")などのOSで使用するための[フリーフォント](../Page/フリーフォント.md "wikilink")として公開している[IPAフォント](../Page/IPAフォント.md "wikilink")もフォントの字形をJIS X 0213:2004のものに変更した\[10\]。

上記対応によりほぼ全てのOSにおいてJIS X 0213:2004の字形が標準で使用可能となった。

JIS X 0213:2004では、JIS X 0213:2004で規定されている全ての文字種が含まれていて、各文字がJIS X 0213:2004での包摂基準に従っており、JIS X 0213:2004で規定されているそれぞれの文字の区別がつけることができる（たとえばJIS X 0213:2004での異体字関係となる「神（片仮名の「ネ」の字形に似た[示偏](../Page/示部.md "wikilink")）」と「（漢字の「示」の字形の示偏）」が違う字形となっている）ことが求められているだけであり、例示字形と完全に同一の字形であることが求められているわけではないので、JIS X 0213:2004の包摂基準範囲内で自由に字形を選択してもJIS X 0213:2004に従った字形（たとえば「辻」が二点之繞でなくてもJIS X 0213に従った字形）ということができる。ただし、一般的にJIS X 0213:2004に従った字形のフォントとして発売されているフォントは、二点之繞や食偏なども含めてJIS X 0213:2004の例示字形に沿った字形を選択しているフォントがほとんどである。

ただし、一部JIS X 0213:2004のコードポイントが割り当てられているにもかかわらず、同じ字形であるがためにUnicodeのコードポイントの全てに対応していないフォントも少なからず存在する。以下にそのケースを挙げる。

  - <span class="Unicode">[Ð](../Page/Ð.md "wikilink")</span> - U+00F0（[アイスランド語](https://ja.wikipedia.org/wiki/アイスランド語 "wikilink")、[フェロー語](../Page/フェロー語.md "wikilink")、[古英語](../Page/古英語.md "wikilink")）
  - <span class="Unicode">[Đ](../Page/Đ.md "wikilink")</span> - U+0111（[クロアチア語](../Page/クロアチア語.md "wikilink")、[サーミ語](../Page/サーミ語.md "wikilink")、[ベトナム語](../Page/ベトナム語.md "wikilink")）
  - <span class="Unicode">[Ɖ](../Page/Ɖ.md "wikilink")</span> - U+0256（[エヴェ語](../Page/エウェ語.md "wikilink")）

以上の3つは、JIS X 0213:2004対応のフォントを使用しても、その全ての表示が保証されるわけではない\[11\]。ただし小文字 (<span class="Unicode">ð, đ, ɖ</span>) に関しては、形状が全て異なるため、JIS X 0213:2004対応フォントで全て表示させることが可能。

## JIS X 0213:2012の改正

[2010年](https://ja.wikipedia.org/wiki/2010年 "wikilink")[11月30日](../Page/11月30日.md "wikilink")の[常用漢字表](https://ja.wikipedia.org/wiki/常用漢字表 "wikilink")改定に対応して、引用例の変更、附属書12の追加がなされた\[12\]。
